## 1. 序列比对

### 1.1 什么是比对以及为什么重要？

**序列比对（sequence alignment）**将两条（或更多）序列上下排列，以识别等价位置：匹配（match）、差异（mismatch）、插入（insertion）和缺失（deletion）。

从生物学角度来看，比对序列允许你：

- 鉴定**同源基因**（homologous genes，共享共同祖先的基因）；
- 检测变异体之间的**突变**；
- 找到可能具有重要功能的**保守区域**；
- 评估生物之间的**进化距离**。

### 1.2 点阵图（Dot Plot）

在讨论复杂算法之前，有一种令人惊讶的简单且强大的可视化工具用于比较两条序列：**点阵图**或 *dot plot*。

#### 工作原理

将两条序列放在一个表格的轴上（一条在水平轴，另一条在垂直轴）。在碱基匹配的每个单元格中标记一个点。完成后，出现的图案揭示了这两条序列之间的关系，**无需进行正式的比对**。

#### 基本示例

让我们比较序列 `ATGCGA`（水平）和 `ATGCGA`（垂直）——即一条序列与其自身：

```text
        A   T   G   C   G   A
  A     •                   •
  T         •
  G             •       •
  C                 •
  G             •       •
  A     •                   •
```

**主对角线**（从左上到右下）完整出现：这确认了序列是相同的。但对角线之外也出现了点——这些揭示了**内部重复**（例如，`G` 出现在两个位置，`A` 也是如此）。

#### 可以检测到哪些图案？

Dot plot 的真正威力在于对图案的视觉解读：

| Dot plot 上的图案 | 表示什么 |
|---|---|
| **连续对角线** | 两条序列之间的高度相似性区域 |
| **位移对角线**（平行于主对角线） | 重复序列或复制 |
| **垂直对角线**（反对角线） | 反向互补序列（回文或反向重复） |
| **对角线断裂** | 插入、缺失或分化区域 |
| **多条短对角线** | 分散重复或低复杂度 |
| **无明显图案** | 序列不共享显著相似性 |

#### 使用两条不同序列的示例

让我们比较 `ATGCATGC` 与其自身：

```text
          A   T   G   C   A   T   G   C
    A     •               •
    T         •               •
    G             •               •
    C                 •               •
    A     •               •
    T         •               •
    G             •               •
    C                 •               •
```

这里出现了**两条平行的对角线**：主对角线和另一条正好位移 4 个位置的对角线。这立即揭示了序列包含一个**串联重复**（`ATGC` 重复两次），仅看序列文本这一点不太明显。

#### 在实践中的用途

Dot plot 特别适用于：

- **将基因组或基因与其自身比较** → 检测重复、复制和回文。
- **比较两个相关的基因组** → 观察保守区域、染色体重排或倒位。
- **正式比对前的初始探索** → 快速了解有多少相似性以及在哪里。
- **可视化复杂结果** → 两个细菌基因组的 dot plot 可以一目了然地揭示它们是共线的还是发生了重排。

#### 窗口和阈值（过滤）

在实践中，逐碱基的 dot plot 会产生大量噪声（不代表真实相似性的随机点）。为了过滤这种噪声，使用**滑动窗口（window）**：不标记每个单独的匹配，仅当大小为 `W` 的窗口中至少有 `X` 个碱基匹配时才标记一个点。

例如，窗口 `W=5` 且阈值 `X=4`：
- 评估 5 个碱基的块；
- 仅当 5 个中至少有 4 个匹配时才标记一个点。

这消除了随机匹配，只留下代表**真实相似性**的对角线。

#### 工具

- **Dotlet**（Web，用于短序列）：https://dotlet.vital-it.ch/
- **Gepard**（Java，用于完整基因组）：https://cube.univie.ac.at/gepard
- **D-Genies**（Web，用于大型基因组比较）：https://dgenies.toulouse.inra.fr/

> [!TIP]
> Dot plot 不能告诉你*哪个*是最佳比对，但它给出了两条序列之间关系的**全局视图**，这是任何单独比对都无法提供的。就像在走进街道之前先看地图。

### 1.3 全局比对 vs. 局部比对

这是一个根本性的区别：

#### 全局比对（Global Alignment）

**从头到尾**比较序列，尝试对齐其整个长度。

> [!TIP]
> **何时使用？** 当两条序列是同一基因或区域且长度相似时。

```text
序列 1:  A T G C G A T C G
序列 2:  A T G C - A T C A
         * * * *   * * * ·
```

#### 局部比对（Local Alignment）

仅在长度可能非常不同的序列内搜索**最佳相似性区域**。

> [!TIP]
> **何时使用？** 当只有序列的一部分共享相似性时（例如，长蛋白质内的一个结构域，或基因组中的一个保守区域）。

```text
序列 1:  ...xxxATGCGATCGxxx...
序列 2:  ...yyyATGCGATCGyyy...
                **********
          仅报告此区域
```

简而言之：

- **全局** = "让我们从头到尾比较所有的东西"
- **局部** = "让我们找到最佳共享片段"

> [!NOTE]
> BLAST 使用**局部**比对。这就是它为什么能在非常不同的序列之间找到部分相似性。

### 1.4 打分（Scoring）概念

为了判断一个比对是"好"还是"坏"，我们需要一个打分系统（*scoring*）。

#### Match 和 Mismatch

对于 DNA 序列，最简单的做法是分配：
- 每个匹配（*match*）的**奖励**，例如 **+1**；
- 每个差异（*mismatch*）的**罚分**，例如 **-1**。

#### Gap 罚分

**Gap**（`-`）代表插入或缺失（*indel*）。如果我们不惩罚 gap，算法可能会引入不必要的 gap 以最大化匹配。

两种主要模型：
- **线性罚分：** 每个 gap 成本相同（例如，每个 gap **-2**）。
- **仿射罚分（affine penalty）：** 打开 gap 的成本高于扩展它。这更符合生物学实际，因为连续 5 个碱基的缺失是单个进化事件，而不是 5 个独立事件。

#### 替代矩阵（Substitution Matrices，用于蛋白质）

在蛋白质中，并非所有替代都一样。将亮氨酸变为异亮氨酸（二者都是疏水性氨基酸）比将亮氨酸变为天冬氨酸（疏水性→带电）不那么"严重"。**BLOSUM** 和 **PAM** 矩阵通过为每对氨基酸分配不同分数来捕捉这些差异。

对于本模块中的 DNA，我们将使用简单的 match/mismatch/gap 方案。

### 1.5 Needleman-Wunsch 算法（全局比对）

该算法于 1970 年发表，是第一个使用**动态规划（dynamic programming）**精确解决全局比对问题的算法。

#### 什么是动态规划？

这是一种问题求解策略，将大问题分解为更小的子问题，按有组织的方式求解，并存储中间结果以避免重复计算。在比对中，这转化为逐步填充一个**矩阵**。

#### 示例：逐步演示

比对序列 `GCATG` 和 `GATTG`，使用：
- Match: **+1**
- Mismatch: **-1**
- Gap: **-2**

##### 步骤 1：创建矩阵

创建一个表格，行代表一条序列，列代表另一条序列。第一行和列用累积 gap 罚分初始化：

```text
        -     G     C     A     T     G
  -     0    -2    -4    -6    -8   -10
  G    -2
  A    -4
  T    -6
  T    -8
  G   -10
```

##### 步骤 2：填充矩阵

对于每个单元格，计算三个选项中的最大值：

1. **对角线**（来自左上）：对角线单元格 + match/mismatch。
2. **上方**（来自上方单元格）：上方单元格 + gap。
3. **左侧**（来自左侧单元格）：左侧单元格 + gap。

单元格保存三个中的**最佳值**和一个指示其来源的箭头。

[G, G] 单元格示例（第一数据行，第一数据列）：
- 对角线：0 + 1（match G=G）= **1**
- 上方：-2 + (-2) = -4
- 左侧：-2 + (-2) = -4
- 结果：**1**（来自对角线）

完整矩阵：

```text
        -     G     C     A     T     G
  -     0    -2    -4    -6    -8   -10
  G    -2     1    -1    -3    -5    -7
  A    -4    -1     0     0    -2    -4
  T    -6    -3    -2    -1     1    -1
  T    -8    -5    -4    -3     0     0
  G   -10    -7    -6    -5    -2     1
```

##### 步骤 3：回溯（Traceback）— 重建比对

填充矩阵给出了最佳比对的**分数**（右下角单元格 = **1**），但不直接告诉我们该比对**是什么**。为了重建它，我们需要从最终单元格**回溯（traceback）**到初始单元格，沿着产生每个值的路径。

**如何跟随路径？**

在步骤 2 中，每个单元格通过从三个选项（对角线、上方、左侧）中选择最大值来填充。回溯包括从每个单元格询问：*"我来自我的三个邻居中的哪一个？"*——并移动到那里。每个移动在比对中都有含义：

| 矩阵中的移动 | 比对中的含义 |
|---|---|
| **↖ 对角线**（左上） | 将序列 1 的一个碱基与序列 2 的一个碱基对齐（match 或 mismatch） |
| **↑ 上方** | 序列 1 中的 Gap（序列 2 前进，序列 1 有一个 `-`） |
| **← 左侧** | 序列 2 中的 Gap（序列 1 前进，序列 2 有一个 `-`） |

让我们逐步观看路径。从右下角单元格 [G,G]（值 **1**）开始：

```text
标有回溯路径 (★) 的矩阵：

              -     G     C     A     T     G
        -     ★0   -2    -4    -6    -8   -10
        G    -2    ★1    -1    -3    -5    -7
        A    -4    -1    ★0     0    -2    -4
        T    -6    -3    -2    ★-1    1    -1
        T    -8    -5    -4    -3    ★0     0
        G   -10    -7    -6    -5    -2    ★1  ← 回溯起点
```

**逐步回溯：**

| # | 当前单元格 | 值 | 来自哪里？ | 原因 | 写入比对 |
|---|---|---|---|---|---|
| 1 | [G, G] (行 5, 列 5) | 1 | **↖ 对角线** 来自 [T, T]=0 | 0 + 1 (match G=G) = 1 ✓ | `G` 与 `G` 对齐 |
| 2 | [T, T] (行 4, 列 4) | 0 | **↖ 对角线** 来自 [T, A]=-1 | -1 + 1 (match T=T) = 0 ✓ | `T` 与 `T` 对齐 |
| 3 | [T, A] (行 3, 列 3) | -1 | **↖ 对角线** 来自 [A, C]=0 | 0 + (-1) (mismatch T≠A) = -1 ✓ | `A` 与 `T` 对齐 (mismatch) |
| 4 | [A, C] (行 2, 列 2) | 0 | **↖ 对角线** 来自 [G, G]=1 | 1 + (-1) (mismatch A≠C) = 0 ✓ | `C` 与 `A` 对齐 (mismatch) |
| 5 | [G, G] (行 1, 列 1) | 1 | **↖ 对角线** 来自 [-,-]=0 | 0 + 1 (match G=G) = 1 ✓ | `G` 与 `G` 对齐 |
| 6 | [-, -] (行 0, 列 0) | 0 | **结束** | 到达原点 | — |

> [!WARNING]
> **为什么在第 2 步中沿对角线走向 [T,A]=-1 而不是向上走向 [T,T]=1？**
>
> 这是学习回溯时最常见的疑问，完全在意料之中。矩阵的该区域看起来像这样：
>
> ```text
>                       col3(A)  col4(T)
>         row 3 (T):    -1       1     ← "为什么不走这里 (↑)?"
>         row 4 (T):    -3      [0]    ← 当前回溯单元格
> ```
>
> 诱惑是向**上**走向 1，因为它是最高值。但回溯**不**寻找值最高的邻居。它寻找**生成了当前单元格值的邻居**。
>
> 当前单元格值为 **0**。让我们看看哪个邻居产生 0：
>
> | 选项 | 邻居 | 邻居的值 | 操作 | 结果 | 等于 0 吗？ |
> |---|---|---|---|---|---|
> | ↖ 对角线 | [T, A] | **-1** | -1 + 1 (match T=T) | **0** | ✅ 是 |
> | ↑ 上方 | [T, T] | **1** | 1 + (-2) (gap) | **-1** | ❌ 否 |
> | ← 左侧 | [T, A] | **-3** | -3 + (-2) (gap) | **-5** | ❌ 否 |
>
> 只有对角线产生 0。向上会产生 -1，而不是 0。因此，单元格 [T,T]=0 **不可能是** 从 [T,T]=1 生成的。
>
> **如果我们强制向上会发生什么？** 我们将获得一个带有 gap 的比对，分数更差：
>
> ```text
> 如果向上 → Seq1 (GCATG) 中的 gap：
>
> Seq 1:   G  C  A  T  -  G      ← Seq2 的 T 不与 Seq1 的任何东西对齐
> Seq 2:   G  ?  ?  ?  T  G
>
> 此路径会产生额外的 gap，分数 ≤ 0，比正确比对的分数 1 更差。
> ```
>
> **关键规则：** 回溯不跟随"值最高的邻居"，而是跟随在应用相应操作（match/mismatch/gap）后**精确重现**当前单元格值的那个邻居。

**构建比对（从后往前）：**

由于回溯从终点走向起点，比对是反向构建的，然后反转：

```text
Seq 1 (GCATG):   G  C  A  T  G
Seq 2 (GATTG):   G  A  T  T  G
                 *  ·  ·  *  *

分数：+1 -1 -1 +1 +1 = 1  ✓（与右下角单元格一致）
```

**如果出现平局怎么办？**

在某些情况下，两个或更多邻居给出相同的最大值。当这种情况发生时，存在**多条可能的路径**，因此存在**多个具有相同分数的最优比对**。回溯选择**一条**路径；如果出现平局，它们之间的选择是任意的（程序决定）。两种比对同样有效。

> [!IMPORTANT]
> 重要的不是记住数字，而是理解逻辑：每个单元格代表**到该点为止对齐这两条子序列的最佳方式**，回溯则逐步重建该最优解。

### 1.6 Smith-Waterman 算法（局部比对）

1981 年发表，它是 Needleman-Wunsch 的修改版，有两个关键区别：

| 特征 | Needleman-Wunsch（全局） | Smith-Waterman（局部） |
|---|---|---|
| **初始化** | 第一行和列使用累积罚分 | 第一行和列使用**零** |
| **负值** | 允许 | 替换为 **0**（永不低于零） |
| **回溯** | 从**右下角**单元格开始 | 从矩阵中**值最高**的单元格开始 |
| **回溯结束** | 到达单元格 [0,0] | 到达一个 **0** 时停止 |

使用相同的序列 `GCATG` 和 `GATTG`（match=+1, mismatch=-1, gap=-2）：

```text
        -     G     C     A     T     G
  -     0     0     0     0     0     0
  G     0     1     0     0     0     1
  A     0     0     0     1     0     0
  T     0     0     0     0     2     0
  T     0     0     0     0     1     1
  G     0     1     0     0     0     2
```

注意与 Needleman-Wunsch 矩阵的区别：
- 第一行和列是**零**（不是累积罚分）。
- 没有负值：任何会产生负数的计算都被替换为 **0**。

#### 逐步回溯

整个矩阵中最高值为 **2**，出现在两个单元格：[T, T]（行 3, 列 4）和 [G, G]（行 5, 列 5）。两者都是有效的回溯起点。让我们走单元格 [T, T]（值 2，行 3, 列 4）：

```text
标有回溯路径 (★) 的矩阵：

              -     G     C     A     T     G
        -     0     0     0     0     0     0
        G     0     1     0     0     0     1
        A     0     0     0    ★1     0     0
        T     0     0     0     0    ★2     0   ← 回溯起点
        T     0     0     0     0     1     1
        G     0     1     0     0     0     2
```

| # | 当前单元格 | 值 | 来自哪里？ | 原因 | 写入比对 |
|---|---|---|---|---|---|
| 1 | [T, T] (行 3, 列 4) | 2 | **↖ 对角线** 来自 [A, A]=1 | 1 + 1 (match T=T) = 2 ✓ | `T` 与 `T` 对齐 |
| 2 | [A, A] (行 2, 列 3) | 1 | **↖ 对角线** 来自 [G, C]=0 | 0 + 1 (match A=A) = 1 ✓ | `A` 与 `A` 对齐 |
| 3 | [G, C] (行 1, 列 2) | 0 | **停止** | 到达一个 **0** → 局部比对结束 | — |

得到的局部比对（反向读取）：

```text
Seq 1 (片段):   A  T
Seq 2 (片段):   A  T
                *  *

分数：+1 +1 = 2  ✓
比对区域：Seq 1 (GCATG) 的位置 3–4 与 Seq 2 (GATTG) 的位置 2–3
```

#### 如果我们走另一个值为 2 的单元格呢？

从 [G, G]（行 5, 列 5，值 2）开始：

```text
标有替代路径 (★) 的矩阵：

              -     G     C     A     T     G
        -     0     0     0     0     0     0
        G     0     1     0     0     0     1
        A     0     0     0     1     0     0
        T     0     0     0     0     2     0
        T     0     0     0     0    ★1     1
        G     0     1     0     0     0    ★2   ← 起点
```

| # | 当前单元格 | 值 | 来自哪里？ | 原因 | 写入 |
|---|---|---|---|---|---|
| 1 | [G, G] (行 5, 列 5) | 2 | **↖ 对角线** 来自 [T, T]=1 | 1 + 1 (match G=G) = 2 ✓ | `G` 与 `G` 对齐 |
| 2 | [T, T] (行 4, 列 4) | 1 | **↖ 对角线** 来自 [T, A]=0 | 0 + 1 (match T=T) = 1 ✓ | `T` 与 `T` 对齐 |
| 3 | [T, A] (行 3, 列 3) | 0 | **停止** | — | — |

替代局部比对：

```text
Seq 1 (片段):   T  G
Seq 2 (片段):   T  G
                *  *

分数：+1 +1 = 2  ✓
比对区域：Seq 1 的位置 4–5 与 Seq 2 的位置 4–5
```

两种局部比对具有相同的分数（2），且两者都是正确的。每种都识别了这两条序列之间**不同的**完美相似性区域。

#### 直接比较：Needleman-Wunsch vs. Smith-Waterman

使用相同的序列和相同的打分，观察两种算法如何产生非常不同的结果：

```text
NEEDLEMAN-WUNSCH (全局)：

Seq 1:   G  C  A  T  G          对齐所有内容，从头到尾
Seq 2:   G  A  T  T  G          包含 mismatch（C≠A, A≠T）
         *  ·  ·  *  *
Score: 1

SMITH-WATERMAN (局部)：

Seq 1:   G  C [A  T] G          处理完整序列，
Seq 2:   G  - [A  T] T  G       但仅报告最佳区域（标有[]）
               *  *
Score: 2
```

> [!WARNING]
> **重要澄清：** Smith-Waterman **不忽略**比对区域以外的序列——算法**确实遍历完整序列**以填充整个矩阵。发生的情况是回溯从矩阵中的最高值开始并在到达零时停止，因此仅**提取**最大相似性的片段。该片段以外的区域对分数没有贡献。
>
> 就像在一本书中搜索一个关键段落：你阅读整本书才能找到它，但最后只划线和报告相关内容。

当序列很短时（如此示例），差异看起来很小。但对于长且不同的序列，局部比对的优势变得明显：

```text
使用长序列的示例：

Seq 1:  ──────────────────── ATGCGATCG ─────────────────────  (2,000 bp)
Seq 2:  ──────── ATGCGATCG ─────────────────────────────────  (5,000 bp)

NEEDLEMAN-WUNSCH: 将强制对齐 2,000 bp 对 5,000 bp，
                   包括带有 gap 和 mismatch 的巨大区域。
                   Score: 可能为负。

SMITH-WATERMAN:   找到共享区域 ATGCGATCG，
                   将其报告为最佳局部比对。
                   Score: +9 (9 个 match)
```

| 方面 | Needleman-Wunsch | Smith-Waterman |
|---|---|---|
| **输入** | 完整序列 | 完整序列（相同） |
| **矩阵** | 完整填充 | 完整填充（相同） |
| **回溯** | 从右下角到 [0,0] | 从**最高值**到到达一个 **0** |
| **结果** | 从头到尾的比对 | 仅**最大相似性区域** |
| 分数 | 1（因 mismatch 而罚分） | 2（仅好的区域） |
| 区域外的 Mismatch | 强制包含它们 | 不报告它们（在回溯之外） |
| **最适合** | 比较同一基因/区域的序列 | 搜索结构域、基序或保守区域 |

> [!NOTE]
> 区别微妙但强大：两种算法都处理完整的序列，但 Needleman-Wunsch 强制从头到尾对齐所有内容，而 Smith-Waterman 仅提取最佳共享片段。这就是为什么 BLAST（使用局部比对）可以在一个巨大蛋白质内找到一个保守结构域，而全局比对将强制也对齐不相关的区域。

### 1.7 同一性（Identity）vs. 相似性（Similarity）

这些概念是不同的，不应作为同义词使用：

- **同一性（Identity）：** 比对中**完全相同**位置的百分比。
- **相似性（Similarity）：** 也考虑**保守替代（conservative substitutions）**的相似程度，在蛋白质中特别重要。

在 **DNA** 中，通常讨论**同一性**，因为碱基之间没有可比的化学性质（一个 A 不"像"一个 G，就像亮氨酸像异亮氨酸那样）。

在**蛋白质**中，**相似性**可能比同一性信息量更大，因为两个不同的氨基酸如果共享化学性质，可能发挥相似的功能。

### 1.8 从理论到实践：与 BLAST 的关系

现在你可以理解为什么 BLAST 快速但不精确：

- **Smith-Waterman**（局部，精确）检查**所有**可能的位置 → 对大型数据库速度慢。
- **BLAST**（局部，启发式）使用种子直接跳到有希望的区域，然后扩展 → 快得多，但可能丢失弱相似性。

在实践中，BLAST 对大多数搜索来说足够了。Smith-Waterman 保留给需要最大灵敏度的案例。

## 3. 使用 BLAST 进行相似性搜索

### 3.1 什么是 BLAST 以及为什么它如此重要？

**BLAST**（*Basic Local Alignment Search Tool*）是生物信息学中用于在数据库中搜索相似序列的最广泛使用的工具。它由 Altschul *等人*于 1990 年发表，自那以后已成为生物信息学领域的网络搜索引擎等价物：你给它一条序列，BLAST 返回数据库中与它最相似的序列。

> [!NOTE]
> **类比：** 想象你有一句用未知语言写的短语，你想在一个拥有数百万本书的图书馆中找到相似的短语。你不能读完整本书，所以你先搜索匹配的关键词，然后在这些匹配周围扩展比较，最后评估哪些是真正的相似性，哪些是偶然。

BLAST 正是这样工作的。

### 3.2 BLAST 的概念性工作原理

BLAST 不逐碱基地将你的序列与数据库中的所有序列进行比较（那太慢了）。相反，它使用三阶段的**启发式策略**：

#### 阶段 1 — 种子（Seeding）

BLAST 将你的序列分成短片段（称为"词"或 *words*），并在数据库中搜索精确或近似匹配。这些初始匹配称为**种子（seeds）**。

#### 阶段 2 — 扩展（Extension）

从每个种子开始，BLAST 向两个方向**扩展比对**，计算分数。如果分数上升，继续扩展。如果下降太多，停止。

#### 阶段 3 — 统计评估（Statistical Evaluation）

扩展的比对被评估以确定它们是否**统计上显著**，或者是否可能偶然发生。

> [!IMPORTANT]
> 这种策略使 BLAST 非常快，但不能保证在所有情况下都能找到最优比对。为此，存在像 Smith-Waterman 这样的精确算法，我们在前面看到过。

### 3.3 BLAST 类型

根据你拥有的序列类型（*query*）和你想搜索的数据库类型（*subject*），BLAST 提供不同的程序：

| 程序 | Query | 数据库 | 何时使用？ |
|---|---|---|---|
| **blastn** | 核苷酸 | 核苷酸 | 搜索相似基因，通过 16S rRNA 鉴定物种 |
| **blastp** | 蛋白质 | 蛋白质 | 搜索同源蛋白质，注释功能 |
| **blastx** | 核苷酸（翻译） | 蛋白质 | 你有 DNA，想搜索蛋白质数据库 |
| **tblastn** | 蛋白质 | 核苷酸（翻译） | 使用蛋白质 query 在未注释基因组中搜索基因 |
| **tblastx** | 核苷酸（翻译） | 核苷酸（翻译） | 双向翻译比较（较慢，较少使用） |

### 3.4 结果解读

当 BLAST 返回结果时，有四个值你需要学会解读：

| 参数 | 测量什么 | "好"值意味着什么 |
|---|---|---|
| **E-value** | 在同等大小的数据库中偶然找到该比对的概率 | 越低越好。E-value < 1e-5 通常被认为显著 |
| **Score（bit score）** | 比对的归一化分数 | 越高越好 |
| **% Identity** | 比对中相同位置的百分比 | 取决于生物学背景 |
| **Query coverage** | 你的序列有多少百分比被对齐 | 低 coverage 可能表示只有一小部分相似 |

> [!WARNING]
> **常见错误：** 低 E-value 且 coverage 为 10% 可能意味着只有一个小结构域是相似的，而不是整个蛋白质是同源的。始终同时查看四个参数。

### 3.5 实践中的 BLAST

BLAST 可以通过两种主要方式使用：

- **BLAST Web** 在 https://blast.ncbi.nlm.nih.gov/ — 适合快速和探索性搜索。
- **命令行 BLAST** — 适合自动化搜索、大数据量或本地数据库。

在本模块中，我们将主要使用 Web 版本。命令行版本将在后续模块中需要时介绍。

## 4. 引物设计和*in silico* PCR

### 4.1 什么是引物，为什么其设计重要？

**引物（primer）**是一个短的 DNA 寡核苷酸（约 18–25 nt），与模板序列结合并允许在 PCR 中启动复制。引物设计是碱基互补性和比对概念最直接的应用之一。

设计一个好的引物是一个生物信息学问题：仅仅选择任何短序列是不够的。必须验证它是特异性的，不会形成二级结构，并且在 PCR 条件下能良好工作。

### 4.2 良好引物的关键参数

| 参数 | 推荐值 | 为什么？ |
|---|---|---|
| **长度** | 18–25 nt | 特异性和结合效率之间的平衡 |
| **GC 含量** | 40–60% | 引物-模板双链的稳定性 |
| **Tm** | 55–65 °C | 引物对应具有相似的 Tm（差值 ≤ 2 °C） |
| **3' 端** | 避免错配；有时使用 "GC clamp" | 对聚合酶延伸至关重要 |
| **自互补性** | 低 | 避免发夹结构和引物二聚体 |
| **扩增子大小** | 100–500 bp（常规 PCR） | 过长产物更难扩增 |

### 4.3 设计工具

- **Primer-BLAST**（https://www.ncbi.nlm.nih.gov/tools/primer-blast/）：设计引物并针对数据库评估特异性。
- **Primer3**：提出具有可配置约束的引物对的经典工具。

### 4.4 *In silico* PCR 和 *in silico* 电泳

***In silico* PCR** 包括通过计算验证一对引物是否会扩增预期产物。这包括：

- 检查**预期的扩增子大小**；
- 评估是否存在**非特异性产物**；
- 预测结果在**电泳凝胶**上的表现（*in silico* 电泳）。
