# 模块 10：代谢网络建模 — 从基因组到生物工厂设计

## 引言

在之前的模块中，你学会了从数据库获取序列、比对它们、重建基因组、注释它们并比较它们。但还有一个问题我们尚未回答：**我们如何告诉细胞该生产什么？**

微生物不仅仅是基因的载体：它们是生化机器，将营养物质转化为能量、生物质和有价值的产品。代谢工程旨在重新编程这些机器，使它们产生感兴趣的化合物：抗生素、工业酶、生物燃料、可生物降解塑料、功能性食品。

为了以理性的方式做到这一点——而非盲目的试错——我们需要代谢的数学表示。这正是**基因组规模代谢模型（GEM, Genome-scale Metabolic Model）**：将基因组转化为方程，使我们能够模拟、预测和设计细胞行为。

本模块连接了你所学的一切：

1. **中心法则及超越** — 从序列到代谢表型。
2. **时空中的生物学** — 为什么细胞是一个动态系统。
3. **数学抽象** — 我们如何简化复杂性以便建模。
4. **作为图的代谢网络** — 节点、边和通量（flux）。
5. **常微分方程（ODE）** — 模拟反应速率。
6. **基因组规模的 COBRA 模型** — 从基因组到化学计量矩阵。
7. **通量平衡分析（FBA, Flux Balance Analysis）** — 在稳态下优化通量。
8. **带有酶约束的模型：GECKO** — 纳入真实的催化能力。

> [!NOTE]
> 本模块要求你已掌握基因组注释的概念（模块 6）并熟悉 Python 基础（模块 2），因为实践使用 **COBRApy** 在 Google Colab 中进行。

## 1. 分子生物学的中心法则 — 及超越

### 1.1 你已经知道的

```text
DNA  ──复制──▶  DNA
DNA  ──转录──▶  RNA  ──翻译──▶  蛋白质
```

### 1.2 全景

遗传信息并不终止于蛋白质。蛋白质催化**化学反应**，转化代谢物、产生能量并制造生长所需的生物质：

```text
DNA ──▶ mRNA ──▶ 蛋白质 ──▶ 酶 ──▶ 生化反应
                                        │
                                  代谢物（葡萄糖、ATP、氨基酸、脂质等）
                                        │
                                  细胞表型：生长、生产、环境响应
```

> [!IMPORTANT]
> 🔑 基因组不仅"存储信息"：它是细胞代谢机器的**操作配方**。每个带有 EC 编号（Enzyme Commission number）注释的基因都可能成为代谢网络中的一个节点。

## 2. 时空中的生物学

细胞不是静态快照：它是一个**动态**系统。代谢物浓度每秒都在变化，取决于：

- 培养基中**可用的营养物质**
- 当时的**基因表达**状态
- **物理条件**（温度、pH、渗透压）
- **培养历史**（延滞期、指数期、稳定期）

这意味着要理解或预测细胞行为，只知道其基因组是不够的。我们需要一个能够捕捉**浓度如何随时间变化**的模型。这把我们引向数学。

---

## 3. 数学抽象以理解细胞

| 抽象层次 | 描述 | 示例 |
|---|---|---|
| **化学计量（Stoichiometric）** | 存在哪些反应以及它们如何连接 | 矩阵 S |
| **动力学（Kinetic）** | 每个反应以何种速率发生 | 使用 Michaelis-Menten 的 ODE |
| **热力学（Thermodynamic）** | 反应的有利程度 | 反应的 ΔG |
| **调控（Regulatory）** | 根据条件哪些基因是活跃的 | 基因调控网络（GRN） |
| **基因组规模** | 生物的所有基因和反应被整合 | GEM (Genome-Scale Model) |

建模的关键在于**我们不需要完美了解一切**：使用正确的抽象层次，我们可以用合理的信息获得有用的预测。

## 4. 代谢网络：图、节点和边

### 4.1 作为图的代谢

代谢可以自然地表示为一个**图（graph）**：每个**节点**代表一个代谢物或化合物，每条**边**代表一个酶催化的反应，通过那些边的**通量（flux）**代表反应发生的速率。

### 4.2 KEGG 代谢图谱

如果你曾见过 KEGG 的完整代谢图谱，你会注意到它看起来像一张无限的蜘蛛网——实际上它是一个非常密集的图。基因组规模代谢模型（GEM）是该图的**形式化数学表示**。

> [!TIP]
> 在以下网址探索真实的代谢图谱：https://www.kegg.jp/kegg/pathway.html

## 5. 常微分方程（ODE）用于模拟代谢

### 5.1 背景中的 ODE

**常微分方程（ODE, Ordinary Differential Equation）**描述一个量如何随时间变化。在生物学中，它们用于模拟代谢物浓度如何变化：

$$\frac{d[X]}{dt} = \sum \text{产生速率} - \sum \text{消耗速率}$$

### 5.2 简单示例：葡萄糖 → 乙醇

```text
简化反应：
  1. 葡萄糖 (G)  ──(v₁)──▶  2 丙酮酸 (P)
  2. 丙酮酸 (P)  ──(v₂)──▶  乙醇 (E) + CO₂
```

ODE 系统：
$$\frac{d[G]}{dt} = -v_1 \quad \frac{d[P]}{dt} = 2v_1 - v_2 \quad \frac{d[E]}{dt} = v_2$$

每种速率 $v$ 使用 **Michaelis-Menten** 动力学建模：

$$v = \frac{V_{max} \cdot [S]}{K_m + [S]}$$

> [!NOTE]
> ODE 捕获动力学过程的**完整时间动态**。例如，丙酮酸在开始时短暂积累，在约 0.5 小时达到峰值，然后在几小时内被消耗。这种瞬态行为正是动力学建模相较于稳态方法的优势所在。

### 5.3 扩展问题：从 3 到 3,000 个反应

一株模型菌如*大肠杆菌*大约有 **2,500 个已注释的代谢反应**。求解 2,500 个 ODE 的联立系统需要：
- 了解**每种**酶的动力学参数（Vmax, Km）；
- 测量**所有**代谢物的初始浓度；
- 在每个时间步长进行数值积分。

实际上，大多数酶的动力学参数**极难测量**。这正是基因组规模模型以不同方法进入的地方。

## 6. 基因组规模代谢模型（GEM）和 COBRA 重建

### 6.1 核心思想：从基因组到矩阵

COBRA 模型（*COnstraints-Based Reconstruction and Analysis*）不要求动力学参数，而是基于一个强大的简化：它们假设**稳态（steady state）**。

在稳态下：$$\frac{d[X]}{dt} = 0 \quad \text{对于所有内部代谢物}$$

在此假设下，ODE 系统简化为**线性代数**问题：

$$\mathbf{S} \cdot \mathbf{v} = \mathbf{0}$$

其中 $\mathbf{S}$ 是维度为 $m \times n$ 的化学计量矩阵（$m$ 个代谢物，$n$ 个反应），$\mathbf{v}$ 是 $n$ 个反应的通量向量。

### 6.2 化学计量矩阵 S

```text
示例，含 4 个代谢物和 4 个反应：

反应：          R1         R2         R3         R4
              Glc→G6P    G6P→F6P    F6P→Pyr    Pyr→ATP

     Glc   │   -1         0          0          0   │
     G6P   │    1        -1          0          0   │   =  S
     F6P   │    0         1         -1          0   │
     Pyr   │    0         0          1         -1   │

惯例：+1 表示代谢物被该反应产生，-1 表示被消耗
```

> [!IMPORTANT]
> 矩阵 $\mathbf{S}$ 的每**行**精确对应于 ODE 系统的一个**微分方程**。假设稳态（$d[X]/dt = 0$）将动态问题转化为静态问题。这消除了对动力学参数的需求，并允许在基因组规模上工作。

### 6.3 从基因组到模型：重建过程

| 步骤 | 名称 | 输入 → 输出 |
|---|---|---|
| 1 | 基因组注释 | 基因组(.fasta) → 带 EC 编号的基因 |
| 2 | 反应分配 | EC 编号 → BiGG/KEGG 中的反应 |
| 3 | GPR 规则 | 基因 → 布尔规则（例如：(b0001 OR b0002)） |
| 4 | 通量界限 | 可逆性 → 界限 [lb, ub] |
| 5 | 交换反应 | 细胞-培养基边界 |
| 6 | 目标函数 | 生物质（biomass） |
| 7 | 手动审编（curation） | 与实验数据对照验证 |

> [!TIP]
> 经过审编的 GEM 模型可在 **BiGG Models**（http://bigg.ucsd.edu/）获取，涵盖数百种生物。审编通常结合自动草案（使用 **CarveMe** 或 **RAVEN**，15-30 分钟）与手动精炼（使用 **COBRApy** 或 **COBRA Toolbox**，数天到数周）。

## 7. 通量平衡分析（FBA, Flux Balance Analysis）

### 7.1 优化问题

由于 S·v = 0 有无穷多个可能解（系统欠定），FBA 增加了两个额外元素：

1. **不等式约束**：通量不能超过某些界限（$\mathbf{lb} \leq \mathbf{v} \leq \mathbf{ub}$）。
2. **目标函数**：最大化或最小化特定反应（通常是生长速率）。

FBA 被表述为一个**线性规划（LP, Linear Program）**：

$$\max_{\mathbf{v}} \quad \mathbf{c}^\top \mathbf{v}$$

$$\text{subject to:} \quad \mathbf{S} \cdot \mathbf{v} = \mathbf{0}, \quad \mathbf{lb} \leq \mathbf{v} \leq \mathbf{ub}$$

### 7.2 FBA 告诉我们什么？

FBA 给出在施加条件下最大化生物学目标的**最优通量分布**：

```text
在基本葡萄糖培养基中对大肠杆菌进行 FBA 模拟：

  条件：葡萄糖摄取 = 10 mmol/gDW/h，好氧
  目标：最大化生长速率 (μ)
  
  结果：μ_opt = 0.874 h⁻¹
  选定通量：糖酵解 10.0，柠檬酸循环 6.0，氧化磷酸化 7.5
```

### 7.3 FBA 的应用

| 分析 | 问题 | 方法 |
|---|---|---|
| **生长预测** | 菌株能在此培养基中生长吗？ | 最大化 μ |
| **基因敲除（Knockout）** | 如果我们删除这个基因会发生什么？ | 对该反应设置 v = 0 |
| **代谢物生产** | 可以生产多少产品？ | 最大化产品通量 |
| **基因必需性** | 哪些基因对生长是必需的？ | 系统性的单基因删除 |
| **OptKnock** | 删除哪些基因以强制生产？ | 双层线性规划 |

### 7.4 FBA 的局限性及其变体

| FBA 假设 | 局限性 | 解决它的变体 |
|---|---|---|
| 稳态 | 不能捕捉时间动态 | **dFBA**（动态 FBA） |
| 所有酶同样可用 | 不反映真实的催化能力差异 | **GECKO / ecGEM** |
| 目标 = 最大化 μ | 在压力下细胞并不总是最大化生长 | **MOMA**, **ROOM** |
| 无显式基因调控 | 不能捕捉根据环境的基因诱导/抑制 | **rFBA**, **iFBA** |

## 8. 带有酶约束的模型：GECKO

### 8.1 酶容量问题

在经典 FBA 中，对通量的唯一约束是数值界限。但现实中，反应速率取决于：
- 细胞中**酶蛋白的量**（表达水平）；
- 该酶的**工作速度**（kcat：催化周转数）。

GECKO 引入的基本关系是：

$$v_i \leq k_{cat,i} \cdot [E_i]$$

此外，所有酶蛋白的总和不能超过细胞的总蛋白质组容量：

$$\sum_i MW_i \cdot [E_i] \leq P_{total}$$

### 8.2 什么是 GECKO？

**GECKO**（*Genome-scale model with Enzymatic Constraints using Kinetic and Omics data*）扩展 GEM 模型，纳入：

1. 每种反应的 **kcat 值**（来自 BRENDA、sabio-rk 或 DLKcat 预测）。
2. **蛋白质组预算约束**（细胞能合成多少蛋白质是有限制的）。
3. 模型中的**显式蛋白质变量**，允许模拟过表达、下调。

### 8.3 为什么 GECKO 重要？

```text
示例：酿酒酵母（S. cerevisiae）生产琥珀酸

  FBA 预测：               GECKO 预测：           实验：
  产量：3.2 mmol/h         产量：1.8 mmol/h        ~1.9 mmol/h
  生长：0.41 h⁻¹           生长：0.35 h⁻¹           ~0.36 h⁻¹
  
  FBA 高估了，因为它不       GECKO 更精确，因为它纳入
  考虑细胞已经花费蛋白质    了蛋白质组预算是有限的
  用于生长，无法分配"无限"  且在生长和生产之间共享。
  的酶用于生产。
```

## 9. 从基因组到生物工厂：代谢工程靶点预测

### 9.1 DBTL 循环

现代代谢工程遵循迭代循环：

```text
        ┌──────────────────────────────────────────────┐
        │       DBTL 循环（设计-构建-测试-学习）         │
        └──────────────────────────────────────────────┘

   ┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
   │ DESIGN  │────▶│  BUILD  │────▶│  TEST   │────▶│  LEARN  │
   │         │     │         │     │         │     │         │
   │ GEM /   │     │ 基因工程│     │ 培养 +  │     │ 用新数据│
   │ ecGEM   │     │ (CRISPR,│     │ 产量和  │     │ 更新模型│
   │ 预测最  │     │  克隆)  │     │ 生长测量│     │         │
   │ 佳靶点  │     │         │     │         │     │         │
   └─────────┘     └─────────┘     └─────────┘     └─────────┘
```

### 9.2 模型可模拟的干预类型

| 干预 | 建模方式 |
|---|---|
| **基因删除** | 将关联反应的通量设为 v = 0 |
| **过表达** | 提高反应的上限（ub）。在 ecGEM 中：增加 [Eᵢ] |
| **下调** | 部分降低 ub（例如 ub × 0.3） |
| **引入异源基因** | 使用适当的 lb/ub 向模型添加新反应 |

### 9.7 系统性靶点搜索：OptKnock

手动评估所有可能的修饰组合是不可行的。利用**组合优化算法**：

| 算法 | 核心思想 |
|---|---|
| **OptKnock** | 双层 LP：在细胞最大化其自身生长的约束下最大化产物产量 → 找到将生长与生产"耦合"的敲除 |
| **RobustKnock** | 类似 OptKnock 但在生物质通量变化下保证鲁棒性 |
| **OptForce** | 通过比较 WT 与期望状态识别要过表达、低表达或删除的反应 |
| **FSEOF** | 扫描通量空间以找到与产物相关的反应 |

## 10. 概念总结

在本模块中你学到了：

- **代谢**可以表示为由酶促反应连接的代谢物的**网络（图）**；
- **ODE**允许模拟浓度如何随时间变化，但需要难以在基因组规模上测量的动力学参数；
- **GEM 模型**通过假设**稳态**（S · v = 0）并将问题表述为线性规划来解决该局限性；
- **FBA**找到在给定约束下最大化生物学目标的最优通量分布；
- **GECKO**通过纳入真实的酶能力（kcat, 蛋白质组预算）扩展 FBA，获得更真实的预测；
- 整个流程从**基因组**出发：一个注释良好的基因组是任何代谢模型的初始输入。

> [!IMPORTANT]
> 代谢建模不是替代实验：而是指导实验。一个好的模型将实验搜索空间从数百种可能的遗传修饰缩小到少数几个成功概率更高的候选方案，加速代谢工程中的 DBTL 循环。
