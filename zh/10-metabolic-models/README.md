# 模块 10：代谢网络建模 — 从基因组到生物工厂设计

## 引言

在之前的模块中，你学会了从数据库获取序列、比对它们、重建基因组、注释它们并比较它们。但还有一个问题我们尚未回答：**我们如何告诉细胞该生产什么？**

微生物不仅仅是基因的载体：它们是生化机器，将营养物质转化为能量、生物质和有价值的产品。代谢工程旨在重新编程这些机器，使它们产生感兴趣的化合物：抗生素、工业酶、生物燃料、可生物降解塑料、功能性食品。

为了以理性的方式做到这一点——而非盲目的试错——我们需要代谢的数学表示。这正是**基因组规模代谢模型（GEM, Genome-scale Metabolic Model）**：将基因组转化为方程，使我们能够模拟、预测和设计细胞行为。

本模块连接了你所学的一切：

1. **中心法则及超越** — 从序列到代谢表型。
2. **时空中的生物学** — 为什么细胞是一个动态系统。
3. **数学抽象** — 我们如何简化复杂性以便建模。
4. **作为图的代谢网络** — 节点、边和通量。
5. **常微分方程（ODE）** — 模拟反应速率。
6. **基因组规模的 COBRA 模型** — 从基因组到化学计量矩阵。
7. **通量平衡分析（FBA）** — 在稳态下优化通量。
8. **带有酶约束的模型：GECKO** — 纳入真实的催化能力。

## 1. 分子生物学的中心法则 — 及超越

### 1.1 你已经知道的

分子生物学的中心法则描述了遗传信息的流动：

```text
DNA  ──复制──▶  DNA
DNA  ──转录──▶  RNA  ──翻译──▶  蛋白质
```

基因编码蛋白质，蛋白质执行生化功能。到此为止是经典的中心法则。

### 1.2 全景

遗传信息并不终止于蛋白质。蛋白质催化**化学反应**，转化代谢物、产生能量并制造生长所需的生物质。完整的流动是：

```text
                   ┌──────────────────────────────────────────────────────┐
                   │               扩展的中心法则                         │
                   └──────────────────────────────────────────────────────┘

    DNA ──▶ mRNA ──▶ 蛋白质 ──▶ 酶 ──▶ 生化反应
                        │                         │
               rRNA, tRNA, miRNA,           代谢物
               snRNA (调控、                (葡萄糖、ATP、
               加工、                      氨基酸、
               蛋白质合成)                 脂质等)
                                                  │
                                          细胞表型：
                                        生长、生产、
                                       环境响应
```

> [!IMPORTANT]
> 🔑 关键图景：基因组不仅"存储信息"：它是细胞代谢机器的**操作配方**。每个带有 EC 编号（Enzyme Commission number）注释的基因都可能成为代谢网络中的一个节点。

## 2. 时空中的生物学

细胞不是静态照片：它是一个**动态**系统。代谢物浓度每秒都在变化，取决于：

- 培养基中**可用的营养物质**（葡萄糖、氧气、氮源）；
- 当时的**基因表达**状态（哪些酶是活跃的）；
- **物理条件**（温度、pH、渗透压）；
- **培养历史**（延滞期、指数期、稳定期）。

这意味着要理解或预测细胞行为，只知道其基因组是不够的。我们需要一个能够捕捉**浓度如何随时间变化**的模型。这把我们引向数学。

---

## 3. 数学抽象以理解细胞

为了建模复杂的生物系统，使用不同的抽象层次：

| 抽象层次 | 描述 | 示例 |
|---|---|---|
| **化学计量（Stoichiometric）** | 存在哪些反应以及它们如何连接 | 矩阵 S |
| **动力学（Kinetic）** | 每个反应以何种速率发生 | 使用 Michaelis-Menten 的 ODE |
| **热力学（Thermodynamic）** | 反应的有利程度 | 反应的 ΔG |
| **调控（Regulatory）** | 根据条件哪些基因是活跃的 | 基因调控网络（GRN） |
| **基因组规模** | 生物的所有基因和反应被整合 | GEM (Genome-Scale Model) |

建模的关键在于**我们不需要完美了解一切**：使用正确的抽象层次，可以用合理的信息获得有用的预测。

## 4. 代谢网络：图、节点和边

### 4.1 作为图的代谢

**图（graph）**是由**节点**（顶点）和**边**（连接）组成的数学结构。代谢可以自然地表示为一个图：

```text
代谢的图表示：

        葡萄糖
           │
    (反应：己糖激酶)
           │
     葡萄糖-6-P ──────────────────────────────────────┐
           │                                            │
   (磷酸葡萄糖                           (葡萄糖-6-P
    异构酶)                             脱氢酶)
           │                                            │
     果糖-6-P                              6-P-葡萄糖酸内酯
           │                                  (磷酸戊糖途径)
           ...

     ○ = 代谢物（节点）
     ─ = 反应（边）
     → = 通量方向
```

每个**节点**代表一个代谢物或化合物。每条**边**代表一个酶催化的反应。通过那些边的**通量**代表反应发生的速率。

如果你曾见过 KEGG（*Kyoto Encyclopedia of Genes and Genomes*）的完整代谢图谱，你会注意到它看起来像无限的蜘蛛网：

```text

║                                                               ║
║   ○──○    ○──○──○       ○       ○──○──○──○──○                 ║
║   │  │    │        ╲   / \     / │                            ║
║   ○  ○──○─○         ○─○   ○──○  ○──○                          ║
║   │     │  \        │     │        │                          ║
║   ○──○──○   ○──○──○─○─────○──○──○──○──○──○                    ║
║       │    /                   │       │                      ║
║       ○──○                     ○──○──○─○──○──○──○             ║
║                                                               ║
║  每个节点 (○) = 代谢物   每条线 (─) = 反应                    ║
╚═══════════════════════════════════════════════════════════════╝
```

> [!TIP]
> 你可以在 https://www.kegg.jp/kegg/pathway.html 探索真实的代谢图谱。

看起来像无限蜘蛛网的东西，实际上是一个非常密集的图。基因组规模代谢模型（GEM）正是该图的**形式化数学表示**。

### 4.3 Transmilenio 类比

一种直观理解代谢网络的方式是将其与城市的公共交通系统进行比较：

```text
╔═══════════════════════╦═══════════════════════════════════╗
║    Transmilenio       ║         代谢网络                  ║
╠═══════════════════════╬═══════════════════════════════════╣
║ 车站                  ║ 代谢物                           ║
║ 公交路线              ║ 酶促反应                         ║
║ 乘客                  ║ 碳/电子流                        ║
║ 公交车容量            ║ 酶的最大活性                     ║
║ Portal 节点           ║ 枢纽代谢物（如 ATP, NADH）       ║
║ 最终目的地            ║ 生物质或目标产物                 ║
║ 阻塞的路线            ║ 被删除/抑制的基因                ║
╚═══════════════════════╩═══════════════════════════════════╝
```

正如在 Transmilenio 中有多条路线从 A 点到 B 点，代谢中也有多种途径将葡萄糖转化为目标产物。建模的任务是在网络当前状态下找到**最优路线**。

## 5. 常微分方程（ODE）模拟代谢

### 5.1 背景中的 ODE

**常微分方程（ODE）**描述一个量如何随时间变化。在生物学中，它们用于模拟代谢物浓度如何变化：

$$\frac{d[X]}{dt} = \sum \text{产生速率} - \sum \text{消耗速率}$$

其中 $[X]$ 是代谢物 X 的浓度，$t$ 是时间。

### 5.2 简单示例：葡萄糖 → 乙醇

考虑一个消耗葡萄糖（G）并产生乙醇（E）的酵母细胞：

```text
简化反应：
  1. 葡萄糖 (G)  ──(v₁)──▶  2 丙酮酸 (P)
  2. 丙酮酸 (P)  ──(v₂)──▶  乙醇 (E) + CO₂
```

**ODE 系统：**

$$\frac{d[G]}{dt} = -v_1$$

$$\frac{d[P]}{dt} = 2v_1 - v_2$$

$$\frac{d[E]}{dt} = v_2$$

其中 $v_1$ 和 $v_2$ 是以 $\text{mmol} \cdot \text{gDW}^{-1} \cdot \text{h}^{-1}$ 为单位的反应速率。

每种速率 $v$ 使用 **Michaelis-Menten** 动力学建模：

$$v = \frac{V_{max} \cdot [S]}{K_m + [S]}$$

### 5.2.1 数值示例详解

使用两个反应的真实参数值，逐步求解系统：

**动力学参数：**

| 反应 | Vmax | Km | 描述 |
|---|---|---|---|
| R1 — 葡萄糖 → 2 丙酮酸 | 2.5 mmol/gDW/h | 0.5 mmol/L | 简化糖酵解复合体 |
| R2 — 丙酮酸 → 乙醇 + CO₂ | 5.0 mmol/gDW/h | 0.2 mmol/L | 丙酮酸脱羧酶 + 乙醇脱氢酶 |

**初始条件（t = 0）：**

| 代谢物 | 浓度 | 描述 |
|---|---|---|
| [G]₀ | 10.0 mmol/L | 培养基中可用的葡萄糖 |
| [P]₀ | 0.0 mmol/L | 初始无丙酮酸积累 |
| [E]₀ | 0.0 mmol/L | 初始无乙醇 |

**步骤 1 — 计算 t = 0 时的速率：**

$$v_1(0) = \frac{V_{max,1} \cdot [G]_0}{K_{m,1} + [G]_0} = \frac{2.5 \times 10.0}{0.5 + 10.0} = 2.381 \ \text{mmol/gDW/h}$$

> 高饱和度：因为 $[G]_0 \gg K_{m,1}$，酶几乎以 $V_{max}$ 运行。

$$v_2(0) = \frac{V_{max,2} \cdot [P]_0}{K_{m,2} + [P]_0} = \frac{5.0 \times 0.0}{0.2 + 0.0} = 0.000 \ \text{mmol/gDW/h}$$

> 没有可用的丙酮酸，反应 2 无法发生。

**步骤 2 — 计算 t = 0 时的导数：**

$$\left.\frac{d[G]}{dt}\right|_{t=0} = -v_1 = -2.381 \ \text{mmol/L/h} \quad \rightarrow \text{葡萄糖减少}$$

$$\left.\frac{d[P]}{dt}\right|_{t=0} = 2v_1 - v_2 = +4.762 \ \text{mmol/L/h} \quad \rightarrow \text{丙酮酸积累}$$

$$\left.\frac{d[E]}{dt}\right|_{t=0} = v_2 = 0.000 \ \text{mmol/L/h} \quad \rightarrow \text{尚未产生乙醇}$$

> [!IMPORTANT]
> 🔍 **关于化学计量的澄清：**
>
> 反应 R1 是：**葡萄糖 → 2 丙酮酸**
>
> 这意味着化学计量系数为：
> - 葡萄糖：**-1**（消耗 1 分子）
> - 丙酮酸：**+2**（产生 2 分子）
>
> 因此导数为：
> - $\frac{d[G]}{dt} = (-1) \times v_1 = -2.381$ mmol/L/h
> - $\frac{d[P]}{dt} = (+2) \times v_1 = +4.762$ mmol/L/h
>
> **重要：**在时间步长 Δt = 0.5 h 中，变化不是完整的通量，而是：
> - Δ[G] = (-2.381) × 0.5 = **-1.190** mmol/L（不是 -2.381）
> - Δ[P] = (+4.762) × 0.5 = **+2.381** mmol/L（不是 +4.762）

> [!TIP]
> **🕐 通量、导数和时间步长之间的关系**
>
> 通量的单位是**每小时**：v₁ = 2.381 mmol/gDW/**h**
>
> 导数表示**每 1 小时**的变化率：
> $$\frac{d[G]}{dt} = -2.381 \text{ mmol/L per } \boxed{1 \text{ hour}}$$
>
> 使用 Euler 方法积分时，乘以实际时间步长（Δt）：
>
> | Δt (h) | 计算 | Δ[G] (mmol/L) | 解读 |
> |---|---|---|---|
> | 1.0 | -2.381 × 1.0 | -2.381 | 完整 1 小时 |
> | 0.5 | -2.381 × 0.5 | -1.190 | 半小时（本案例） |
> | 0.1 | -2.381 × 0.1 | -0.238 | 1/10 小时 |
>
> **黄金法则：** 变化 = (导数) × (时间步长)
> $$\Delta [X] = \frac{d[X]}{dt}\bigg|_n \times \Delta t$$

**步骤 3 — 数值积分（Δt = 0.5 h 步长，显式 Euler 方法）：**

**数值结果表：**

| t (h) | [G] (mmol/L) | [P] (mmol/L) | [E] (mmol/L) | v₁ | v₂ |
|---|---|---|---|---|---|
| 0.0 | 10.000 | 0.000 | 0.000 | 2.381 | 0.000 |
| 0.5 | 8.810 | 2.381 | 0.000 | 2.291 | 4.297 |
| 1.0 | 7.665 | 0.537 | 2.149 | 2.185 | 3.302 |
| 1.5 | 6.572 | 0.213 | 3.800 | 2.065 | 2.760 |
| 2.0 | 5.539 | 0.139 | 5.180 | 1.929 | 2.545 |
| 3.0 | 3.595 | 0.104 | 7.853 | 1.608 | 2.282 |
| 4.0 | 1.864 | 0.090 | 9.970 | 1.201 | 2.121 |
| 5.0 | 0.500 | 0.080 | 11.482 | 0.625 | 2.000 |
| 6.0 | 0.032 | 0.030 | 12.710 | 0.140 | 1.429 |
| 7.0 | 0.001 | 0.003 | 13.066 | 0.005 | 0.067 |
| 8.0 | 0.000 | 0.000 | 13.070 | 0.000 | 0.000 |

> [!NOTE]
> 📌 阅读表格：
> - **t = 0 → 0.5 h:** `v₁ ≫ v₂`，因为有大量葡萄糖但尚无丙酮酸。丙酮酸快速积累至约 2.38 mmol/L。
> - **t = 0.5 → 1.5 h:** 现在 `v₂` 急剧上升（有丙酮酸可用，浓度高）。丙酮酸迅速下降，乙醇开始积累。
> - **t > 1.5 h:** 丙酮酸稳定在低水平（约 0.1 mmol/L）：R1 产生的量几乎等于 R2 消耗的量。**这就是 FBA 从 t = 0 开始假设的准稳态（quasi-steady state）。**
> - **t ≈ 8 h:** 所有葡萄糖耗尽。最终乙醇约为 13.07 mmol/L。

**动力学的图形表示：**

```text
[mmol/L]

     │  G ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░░░░░
   8 ┤
     │
   6 ┤                         E  ░░░░░░░░░░░░░░░░▓▓▓▓▓▓▓▓▓▓▓▓
     │
   4 ┤
     │
   2 ┤          P ▲ (峰值约 t=0.5h)
     │              ╲_______________ ≈ 0.1 mmol/L (准稳态)
   0 ┤─────┬─────────┬─────────┬──────────┬─────────▶ t (h)
     0    0.5       2.0       4.0        6.0       8.0

  ▓▓ 葡萄糖 [G] — 刚开始平稳下降（高饱和度），然后耗尽
  ░░ 乙醇 [E] — 滞后于 [G]；有葡萄糖时就积累
     丙酮酸 [P] — 初始瞬时峰值，然后保持低水平的准稳态
```

> [!TIP]
> 丙酮酸在被消耗之前瞬时积累是**动力学行为不明显**的中间代谢物的经典示例。动力学系统（ODE）捕获了这种瞬时行为；FBA 假设 [P] 从 t = 0 开始已经处于稳态。两种方法都有用：ODE 用于理解系统在时间中**如何变化**，FBA 用于在基因组规模上预测**最优终态**，无需了解每种酶的动力学参数。

### 5.3 扩展问题：从 3 到 3,000 个反应

一株模型菌如*大肠杆菌*大约有 **2,500 个已注释的代谢反应**。求解 2,500 个 ODE 的联立系统需要：

- 了解**每种**酶的动力学参数（Vmax, Km）；
- 测量**所有**代谢物的初始浓度；
- 在每个时间步长进行数值积分。

实际上，大多数酶的动力学参数**极难测量**。这正是基因组规模模型以不同方法进入的地方。

## 6. 基因组规模代谢模型（GEM）和 COBRA 重建

### 6.1 核心思想：从基因组到矩阵

COBRA 模型（*COnstraints-Based Reconstruction and Analysis*）不要求动力学参数，而是基于一个强大的简化：它们假设**稳态（steady state）**。

稳态意味着代谢物的细胞内浓度不随时间变化：

$$\frac{d[X]}{dt} = 0 \quad \text{对于所有内部代谢物}$$

这等价于说**进入网络每个节点的量等于出来的量**：每个代谢物上的净通量为零。

在此假设下，ODE 系统简化为**线性代数**问题：

$$\mathbf{S} \cdot \mathbf{v} = \mathbf{0}$$

其中 $\mathbf{S}$ 是维度为 $m \times n$ 的化学计量矩阵（$m$ 个代谢物，$n$ 个反应），$\mathbf{v}$ 是 $n$ 个反应的通量向量，$\mathbf{0}$ 是强加稳态条件的零向量。

### 6.2 化学计量矩阵 S

化学计量矩阵编码了代谢网络的所有信息。它与 ODE 系统的直接关系是：

```text
示例，含 4 个代谢物和 4 个反应：

反应：          R1        R2        R3        R4
              Glc→G6P   G6P→F6P   F6P→Pyr   Pyr→ATP

     Glc  │   -1        0         0         0   │
     G6P  │    1       -1         0         0   │   =  S
     F6P  │    0        1        -1         0   │
     Pyr  │    0        0         1        -1   │

惯例：
• 值  1：代谢物被该反应产生
• 值 -1：代谢物被该反应消耗
• 值  0：代谢物不参与该反应
```

**与 ODE 的关系** — G6P 的行直接告诉我们：

$$\frac{d[\text{G6P}]}{dt} = 1 \cdot v_1 + (-1) \cdot v_2 = v_1 - v_2$$

在稳态下（$d[\text{G6P}]/dt = 0$）：

$$v_1 - v_2 = 0 \implies v_1 = v_2$$

产生的 G6P 量必须恰好等于消耗的量。

> [!IMPORTANT]
> 矩阵 $\mathbf{S}$ 的每**行**精确对应于 ODE 系统的一个**微分方程**。假设稳态（$d[X]/dt = 0$）将动态问题转化为静态问题：$\mathbf{S} \cdot \mathbf{v} = \mathbf{0}$。这消除了对动力学参数的需求，并允许在基因组规模上工作。

### 6.3 从基因组到模型：重建过程

构建 GEM 是一个整合基因组、生化和文献信息的迭代过程：

| 步骤 | 名称 | 输入 → 输出 | 示例 |
|---|---|---|---|

## 7. 通量平衡分析（FBA）

### 7.1 优化问题

由于 S·v = 0 有无穷多个可能解（系统欠定），FBA 增加了两个额外元素：

> [!TIP]
> 用于构建和审编 GEM 模型的相同工具也实现了 FBA 及所有衍生分析：
>
> | 工具 | 语言 | FBA 中的主要功能 | 链接 |
> |---|---|---|---|
> | **COBRApy** | Python | `model.optimize()`, `production_envelope()`, `single_gene_deletion()` | github.com/opencobra/cobrapy |
> | **COBRA Toolbox** | MATLAB | `optimizeCbModel()`, `robustknock()`, `optKnock()`, FVA, MOMA | github.com/opencobra/cobratoolbox |
> | **RAVEN Toolbox** | MATLAB | `solveLP()`, 通量分析，与组学数据集成 | github.com/SysBioChalmers/RAVEN |
>
> 在本实践活动中，我们使用 **COBRApy**，因为它是 Python 且可直接在 Google Colab 中运行。

1. **不等式约束**：通量不能超过某些界限（$\mathbf{lb} \leq \mathbf{v} \leq \mathbf{ub}$）。
2. **目标函数**：最大化或最小化特定反应（通常是生长速率）。

FBA 被表述为一个**线性规划（LP）**：

$$\max_{\mathbf{v}} \quad \mathbf{c}^\top \mathbf{v}$$

$$\text{subject to:} \quad \mathbf{S} \cdot \mathbf{v} = \mathbf{0}$$

$$\mathbf{lb} \leq \mathbf{v} \leq \mathbf{ub}$$

### 7.2 FBA 告诉我们什么？

FBA 给出在施加条件下最大化生物学目标的**最优通量分布**：

```text
在基本葡萄糖培养基中对大肠杆菌进行 FBA 模拟：

  条件：葡萄糖摄取 = 10 mmol/gDW/h，好氧
  目标：最大化生长速率 (μ)
  
  结果：
    μ_opt  = 0.874 h⁻¹
    
  选定通量：
    糖酵解：           10.0  mmol/gDW/h
    柠檬酸循环：        6.0  mmol/gDW/h
    氧化磷酸化：         7.5  mmol/gDW/h
    乙酸排泄：           0.4  mmol/gDW/h
```

### 7.3 FBA 的应用

| 分析 | 问题 | 方法 |
|---|---|---|
| **生长预测** | 菌株能在此培养基中生长吗？ | 最大化 μ |
| **基因敲除（Knockout）** | 如果我们敲除这个基因会发生什么？ | 对该反应设置 v = 0 |
| **代谢物生产** | 可以生产多少产品？ | 最大化产品通量 |
| **基因必需性** | 哪些基因对生长是必需的？ | 系统性的单基因删除 |
| **OptKnock** | 敲除哪些基因以强制生产？ | 双层线性规划 |

```text
示例：敲除 pgi 基因（磷酸葡萄糖异构酶）的 FBA

  正常条件：   μ = 0.874 h⁻¹
  Δpgi (v_pgi = 0):   μ = 0.460 h⁻¹，流向磷酸戊糖途径的通量 ↑↑
  
  解读：该删除将碳重新导向磷酸戊糖途径
  （对生产 NADPH、赤藓糖-4-P、芳香族化合物很有用）。
```

### 7.4 标准 FBA 的局限性及解决它们的变体

FBA 是强大的工具，但具有重要的局限性。表格也指出了为解决它们而存在的变体：

| FBA 假设 | 局限性 | 解决它的变体 |
|---|---|---|
| 稳态 | 不能捕捉时间动态 | **dFBA**（动态 FBA） |
| 所有酶同样可用 | 不反映真实的催化能力差异 | **GECKO / ecGEM** |
| 目标 = 最大化 μ | 在压力或工业条件下，细胞并不总是最大化生长 | **MOMA**, **ROOM** |
| 无显式基因调控 | 不能捕捉根据环境的基因诱导/抑制 | **rFBA**, **iFBA** |

> [!WARNING]
> 生物工厂设计最关键的局限性是 FBA 将所有酶在其通量界限内视为同样可用。现实中，每种酶都有有限的蛋白质量和特定的催化能力（kcat）。忽略这一点可能导致不切实际的乐观生产预测。

### 7.5 动态 FBA（dFBA）：将 ODE 与 FBA 结合

到目前为止描述的一切都对应于**稳态 FBA (ssFBA)**：模型解决某一时刻的通量分布，假设胞外浓度不变化。这对于恒化器培养是合理的，但在**批式培养（batch culture）**中，葡萄糖耗尽、pH 变化、生物质增长：系统*确实*随时间演变。

**动态 FBA (dFBA)** 结合了两者的优点：使用 ODE 系统随时间跟踪胞外浓度，并在每个时间步解决 FBA 以获得在该条件下的最优胞内通量。

#### dFBA 的原理

```text
           t = 0          t = dt         t = 2*dt       ...
              |               |               |
  [Glc]_0 ----|               |               |
  [Biom]_0    |               |               |
  [EtOH]_0    |               |               |
              |               |               |
  +-----------v---------------v---------------v-----------+
  |         外层（胞外 ODE）                               |
  |    培养基中的浓度：Glc, X, EtOH, ...                   |
  +-----------+---------------+---------------+-----------+
              |  通量 v(t)    |               |
              v               v               v
  +-----------------------------------------------------------+
  |     内层（胞内 FBA，每一个 dt 步）                        |
  |     使用当前步的界限 ub(t) 解决 LP                        |
  |     返回：mu(t), v_uptake(t), v_prod(t), ...              |
  +-----------------------------------------------------------+
```

**外层 — 胞外 ODE**（随时间积分）：

$$\frac{d[\text{Glc}]}{dt} = -v_{uptake}(t) \cdot X(t)$$

$$\frac{d[X]}{dt} = \mu(t) \cdot X(t)$$

$$\frac{d[\text{EtOH}]}{dt} = v_{etoh}(t) \cdot X(t)$$

**内层 — 胞内 FBA**（在每个时刻 $t$ 解决）：

$$\max_{\mathbf{v}} \quad \mathbf{c}^\top \mathbf{v} \qquad \text{s.a.} \quad \mathbf{S}\cdot\mathbf{v} = \mathbf{0}, \quad \mathbf{lb} \leq \mathbf{v} \leq \mathbf{ub}(t)$$

通量界限 $\mathbf{ub}(t)$ 每一步都变化，因为它们取决于当前的胞外浓度。例如，葡萄糖摄取受可用浓度的约束，通过 Michaelis-Menten 动力学：

$$v_{\mathrm{uptake}}(t) \leq \frac{v_{\mathrm{uptake}}^{\max} \cdot \mathrm{[Glc]}(t)}{K_s + \mathrm{[Glc]}(t)}$$

#### 示例：模拟葡萄糖培养基中*大肠杆菌*的批式培养

```text
 t (h) │ [Glc] (g/L) │ [X] (g/L) │  [EtOH] (mmol/L) │  μ(t) (h⁻¹)
───────┼─────────────┼───────────┼──────────────────┼──────────────
   0.0 │    10.0     │   0.10    │       0.0        │    0.87
   1.0 │     8.1     │   0.24    │       1.8        │    0.87
   2.0 │     5.6     │   0.57    │       5.2        │    0.85
   3.0 │     2.2     │   1.31    │      11.3        │    0.74
   4.0 │     0.4     │   2.71    │      18.6        │    0.41
   4.5 │     0.0     │   3.10    │      20.1        │    0.00   ← 葡萄糖耗尽
   5.0 │     0.0     │   3.10    │      20.1        │    0.00
```

> [!NOTE]
> 注意 $\mu(t)$ 在最后下降**不是因为代谢网络发生了变化**，而是因为 FBA 接收 $[Glc] \to 0$ 作为输入，这降低了摄取界限，从而降低了预测的生长速率。这就是 dFBA 的优雅之处：时间动态从 ODE 和 FBA 的耦合中涌现出来，无需胞内动力学参数。

#### 比较：ssFBA vs. dFBA vs. 动力学 ODE

| 特征 | ssFBA | dFBA | 动力学 ODE |
|---|---|---|---|
| 预测胞内通量 | ✅ | ✅ | ✅ |
| 捕捉时间动态 | ❌ | ✅ | ✅ |
| 需要动力学参数（$K_m$, $V_{max}$） | ❌ | 仅胞外 | ✅（全部） |
| 扩展到基因组水平（数千个反应） | ✅ | ✅ | ❌ |
| 计算成本 | 低 | 中 | 高 |
| 在 COBRApy / COBRA Toolbox 中实现 | ✅ | ✅（`cobra.flux_analysis.pfba`, `dfba` 包） | `scipy` 库 |

> [!TIP]
> 在 Python 中有 **`dfba`** 包（github.com/biosustain/dfba），由 Novo Nordisk Foundation Center for Biosustainability 开发，它将 COBRApy 与 `scipy.integrate` 集成，以在任何 GEM 模型上直接求解 dFBA 模拟。

## 8. 带有酶约束的模型：GECKO

### 8.1 酶容量问题

在经典 FBA 中，对通量的唯一约束是数值界限 `[lb, ub]`。但现实中，反应的速率取决于：

- 细胞中**酶蛋白的量**（表达水平）；
- 该酶的**工作速度**（kcat：催化周转数）。

GECKO 引入的基本关系是：

$$v_i \leq k_{cat,i} \cdot [E_i]$$

其中 $v_i$ 是反应 $i$ 的通量（mmol·gDW⁻¹·h⁻¹），$k_{cat,i}$ 是酶 $i$ 的催化周转数（转换为 h⁻¹），$[E_i]$ 是可用的酶 $i$ 的量（mmol·gDW⁻¹）。

此外，所有酶蛋白的总和不能超过细胞的总蛋白质组容量：

$$\sum_i MW_i \cdot [E_i] \leq P_{total}$$

其中 $MW_i$ 是酶 $i$ 的分子量（g·mol⁻¹），$P_{total}$ 是可用的蛋白质组分数（g·gDW⁻¹，通常 ≈ 0.5）。

### 8.2 什么是 GECKO？

**GECKO**（*Genome-scale model with Enzymatic Constraints using Kinetic and Omics data*）是 Benjamín Sánchez 等人开发的框架，它扩展 GEM 模型，纳入：

1. 每种反应的 **kcat 值**（来自 BRENDA、sabio-rk 或 DLKcat/TurNuP 预测）。
2. **蛋白质组预算约束**（细胞能合成多少蛋白质是有限制的）。
3. 模型中的**显式蛋白质变量**，允许模拟过表达、下调以及定量蛋白质组学数据。

ecGEM（GECKO）的优化问题扩展了 FBA 的 LP，增加了新的约束：

| | 经典 FBA | ecGEM (GECKO) |
|---|---|---|
| **稳态** | $\mathbf{S} \cdot \mathbf{v} = \mathbf{0}$ | $\mathbf{S}_{ext} \cdot \mathbf{v}_{ext} = \mathbf{0}$ |
| **通量界限** | $\mathbf{lb} \leq \mathbf{v} \leq \mathbf{ub}$ | $\mathbf{lb} \leq \mathbf{v} \leq \mathbf{ub}$ |
| **酶约束** | — | $v_i \leq k_{cat,i} \cdot [E_i]$ |
| **蛋白质组预算** | — | $\sum_i MW_i \cdot [E_i] \leq P_{total}$ |
| **非负性** | — | $[E_i] \geq 0$ |
| **目标** | $\max \ \mathbf{c}^\top \mathbf{v}$ | $\max \ \mathbf{c}^\top \mathbf{v}$ |

### 8.3 为什么 GECKO 重要？

```text
示例：酿酒酵母（S. cerevisiae）生产琥珀酸

  FBA 经典预测：             GECKO 预测：          实验：
  ───────────────            ───────────          ─────
  产量：3.2 mmol/h           产量：1.8 mmol/h       ~1.9 mmol/h
  生长：0.41 h⁻¹             生长：0.35 h⁻¹          ~0.36 h⁻¹
  
  FBA 高估了，因为它不         GECKO 更精确，因为它纳入
  考虑细胞已经花费蛋白质      了蛋白质组预算是有限的
  用于生长，无法分配"无限"    且在生长和生产之间共享。
  的酶用于生产。
```

### 8.4 GECKO 工作流

```text
1. 基础 GEM 模型 (SBML / JSON)
           │
           ▼
2. kcat 数据库
   (BRENDA, sabio-rk, DLKcat, TurNuP)
           │
           ▼
3. [可选] 定量蛋白质组学数据
   (每种条件下有多少蛋白质，以 mg/gDW 计)
           │
           ▼
4. GECKO pipeline (Python)
   → 扩展 S 以包含蛋白质变量 [E_i]
   → 为每个酶促反应分配 kcat
   → 定义总蛋白质组预算
           │
           ▼
5. ecGEM 准备就绪，可进行模拟
   → 带酶约束的 FBA/pFBA
   → 逼真的敲除和过表达模拟
   → 更精确的生物工厂设计预测
```

## 9. 从基因组到生物工厂：代谢工程靶点预测

本节整合所有内容并回答本模块的核心问题：**我们如何使用 GEM 和 ecGEM 模型来决定修改哪些基因、敲除哪些基因以及过表达哪些基因，以使细胞生产我们想要的东西？**

### 9.1 理性设计循环（DBTL）

现代代谢工程遵循迭代循环：

```text
        ┌─────────────────────────────────────────────┐
        │       DBTL 循环（设计-构建-测试-学习）         │
        └─────────────────────────────────────────────┘

   ┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
   │   DESIGN     │────▶│    BUILD     │────▶│     TEST     │────▶│    LEARN     │
   │              │     │              │     │              │     │              │
   │ GEM / ecGEM  │     │ 基因工程     │     │ 培养 +       │     │ 用新数据     │
   │ 预测最优靶点 │     │ (CRISPR,     │     │ 产量和生长   │     │ 更新模型     │
   │              │     │  克隆)       │     │ 测量         │     │              │
   └──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
          ▲                                                               │
          └───────────────────────────────────────────────────────────────┘
```

计算模型在 **DESIGN** 阶段起作用：它使我们能够在接触一根试管之前，在数小时内评估数千种遗传修饰组合。

### 9.2 模型预测的干预类型

GEM 模型可以模拟四种基本类型的遗传干预：

```text
┌──────────────────────┬──────────────────────────────────────────────────────────┐
│ 干预                 │ 建模方式                                                  │
├──────────────────────┼──────────────────────────────────────────────────────────┤
│ 基因删除             │ 将关联反应的通量设为 v = 0                                │
│                      │ （使用 GPR 规则传播效应）                                 │
├──────────────────────┼──────────────────────────────────────────────────────────┤
│ 过表达               │ 提高反应的上限 (ub)                                      │
│                      │ 在 ecGEM 中：增加 [E_i] → 更大的催化能力                 │
├──────────────────────┼──────────────────────────────────────────────────────────┤
│ 通量下调             │ 部分降低 ub（例如 ub × 0.3）                             │
│ (downregulation)     │ 在 ecGEM 中：减少可用的 [E_i]                            │
├──────────────────────┼──────────────────────────────────────────────────────────┤
│ 引入异源基因         │ 使用适当的 lb/ub 向模型添加新反应                         │
│                      │ （来自另一种生物的生物合成途径）                          │
└──────────────────────┴──────────────────────────────────────────────────────────┘
```

### 9.3 策略 1 — 删除：通过消除竞争途径重新导向通量

想法很简单：如果细胞有一条与生产目标化合物相竞争的替代途径，**消除它**可以迫使碳流向我们想要的方向。

```text
示例：大肠杆菌中生产琥珀酸

  简化代谢网络：

  葡萄糖 ──▶ PEP ──▶ 丙酮酸 ──┬──▶ 乙酸   (竞争途径 A)
                              ├──▶ 乙醇   (竞争途径 B)
                              ├──▶ 乳酸   (竞争途径 C)
                              └──▶ OAA ──▶ 苹果酸 ──▶ 延胡索酸 ──▶ 琥珀酸 ✓

  删除策略：
    Δpta  (磷酸转乙酰酶)      → 消除通向乙酸的途径
    Δadhb (乙醇脱氢酶)        → 消除通向乙醇的途径
    Δldha (乳酸脱氢酶)        → 消除通向乳酸的途径

  FBA 预测结果：
    WT:              琥珀酸 = 0.1 mmol/gDW/h,  μ = 0.87 h⁻¹
    Δpta:            琥珀酸 = 0.4 mmol/gDW/h,  μ = 0.75 h⁻¹
    Δpta Δldha:      琥珀酸 = 1.2 mmol/gDW/h,  μ = 0.61 h⁻¹

  解读：
    每次删除将更多碳重新导向琥珀酸，
    但以降低生长为代价（权衡 trade-off）。
```

**如何使用 COBRApy 实现：**

```python
from cobra.flux_analysis import single_gene_deletion, double_gene_deletion

# 系统性单基因删除：评估每个基因
resultados = single_gene_deletion(model)

# 筛选其删除能增加目标产物产量的基因
# （最大化 EX_succ_e 作为目标函数）
with model:
    model.objective = "EX_succ_e"
    for gen_id in genes_a_evaluar:
        with model:
            model.genes.get_by_id(gen_id).knock_out()
            sol = model.optimize()
            produccion = sol.objective_value
```

### 9.4 策略 2 — 过表达：增加限速步骤的容量

有时通量无法到达产物，因为一种酶是途径的**限速步骤（bottleneck）**。过表达它可以增加其容量并可能大幅提高产量。

```text
示例：过表达 PPC（磷酸烯醇丙酮酸羧化酶）用于生产琥珀酸

  PEP + CO₂ ──(PPC)──▶ OAA  ← 直接为琥珀酸途径供料的步骤

  在标准 FBA 中：
    PPC 的原始界限：0 ≤ v_PPC ≤ 1000 mmol/gDW/h
    WT 中预测的通量：  v_PPC = 2.3 mmol/gDW/h

  过表达（在带有 GECKO 的 ecGEM 中）：
    [E_PPC] × 3 → 增加可用酶量
    新通量： v_PPC = 6.8 mmol/gDW/h
    琥珀酸产量：相对于 WT +180%

  在经典 FBA 中（无 ecGEM）：
    使用 ub(PPC) × 3 = 3000 进行模拟 → 精度较低，
    但可以捕捉效应的方向。
```

**如何使用 COBRApy 实现：**

```python
with model:
    # 模拟过表达：提高反应的上限
    rxn = model.reactions.get_by_id("PPC")
    rxn.upper_bound = rxn.upper_bound * 3   # 3x 过表达

    sol = model.optimize()
    print(f"PPC 过表达条件下的产量：{sol.fluxes['EX_succ_e']:.3f}")
```

> [!NOTE]
> 在 **ecGEM (GECKO)** 中，过表达通过增加酶蛋白的量 `[E_i]` 来建模。这更精确，因为它尊重蛋白质组预算：过表达一种酶会消耗其他酶可用的蛋白质预算的一部分。

### 9.5 策略 3 — 通量部分下调（CRISPRi / RNAi）

并非总是方便完全消除一个反应：有时**部分下调**（downregulation）允许在重新导向通量的同时保持生长。这正是 CRISPRi 或 RNAi 敲低在实验中所做的。

```text
示例：大肠杆菌中 PYK（丙酮酸激酶）的部分下调

  PEP ──(PYK)──▶ 丙酮酸 + ATP

  如果完全消除 PYK (Δpyk)：细胞失去 ATP → μ 大幅下降
  如果 PYK 下调至 20%：PEP 积累 → 更多碳可用于 OAA → 琥珀酸 ↑

  不同下调水平的模拟：

  下调 PYK │ v_PYK (mmol/gDW/h) │ 琥珀酸 │ μ (h⁻¹)
  ────────┼────────────────────┼─────────┼────────
  0%  (WT) │        8.5         │   0.1   │  0.87
  50%      │        4.3         │   0.6   │  0.78
  80%      │        1.7         │   1.1   │  0.65
  100% (Δpyk)│      0.0         │   0.8   │  0.31   ← 比 80% 更差！
```

**如何使用 COBRApy 实现：**

```python
niveles_reduccion = [1.0, 0.5, 0.2, 0.1, 0.0]  # 原始 ub 的比例

resultados = []
with model:
    model.objective = "EX_succ_e"
    ub_original = model.reactions.get_by_id("PYK").upper_bound

    for nivel in niveles_reduccion:
        with model:
            model.reactions.get_by_id("PYK").upper_bound = ub_original * nivel
            sol = model.optimize()
            resultados.append((nivel, sol.objective_value))

for nivel, prod in resultados:
    print(f"  PYK 下调至 {nivel*100:.0f}%：琥珀酸 = {prod:.3f} mmol/gDW/h")
```

### 9.6 策略 4 — 引入异源途径

在许多情况下，宿主微生物**没有目标化合物的生物合成途径**。解决方案是引入来自另一种生物的基因。模型允许你在**克隆之前**评估异源途径是否可行。

```text
示例：在大肠杆菌中生产 1,4-丁二醇 (BDO)
         （一种具有多种工业应用的化合物）

  大肠杆菌不天然生产 BDO。
  异源途径（取自多种生物）：

  琥珀酸 ──▶ 琥珀酰-CoA ──▶ 4-HB-CoA ──▶ 4-HB ──▶ BDO
                    ↑                ↑              ↑
                 (SucD from      (4hbD from      (adhE2 from
                Clostridium)  Porphyromonas)   Clostridium)

  建模方式：
    1. 向大肠杆菌的 GEM 模型添加 3 个新反应
    2. 添加代谢物 BDO 及其排泄反应 (EX_bdo_e)
    3. 运行 FBA 最大化 EX_bdo_e
    4. 如果解可行且通量 > 0：该途径可行

  结果：
    无额外优化：           BDO = 0.8 mmol/gDW/h
    + 删除竞争途径（更多琥珀酰-CoA 被导向）：BDO = 2.1 mmol/gDW/h
```

**如何使用 COBRApy 实现：**

```python
from cobra import Reaction, Metabolite

with model:
    # 创建新代谢物
    bdo = Metabolite('bdo_c', formula='C4H10O2', name='1,4-丁二醇', compartment='c')

    # 创建异源途径的最终反应（简化）
    rxn_bdo = Reaction('BDO_synth')
    rxn_bdo.name = 'BDO synthesis (异源)'
    rxn_bdo.lower_bound = 0
    rxn_bdo.upper_bound = 1000
    # 化学计量：4HB → BDO（简化）
    rxn_bdo.add_metabolites({
        model.metabolites.get_by_id('4hb_c'): -1,
        bdo: 1
    })
    model.add_reactions([rxn_bdo])

    # 排泄反应
    rxn_ex = Reaction('EX_bdo_e')
    rxn_ex.add_metabolites({bdo: -1})
    model.add_reactions([rxn_ex])

    # 评估
    model.objective = 'EX_bdo_e'
    sol = model.optimize()
    print(f"异源途径的 BDO 产量：{sol.objective_value:.3f}")
```

### 9.7 系统性靶点搜索：OptKnock 及类似方法

手动评估所有可能的修饰组合是不可行的：对于 1,000 个候选基因，有超过 10¹⁵ 种可能的双敲除组合。为此存在基于模型的**组合优化算法**：

```text
菌株设计算法：

┌────────────────┬────────────────────────────────────────────────────────┐
│ 算法           │ 核心思想                                               │
├────────────────┼────────────────────────────────────────────────────────┤
│ OptKnock       │ 双层 LP：在细胞最大化其自身生长的约束下                │
│                │ 最大化产物产量                                          │
│                │ → 找到将生长与生产"耦合"的敲除                          │
│                │   （细胞为了生长而生产）                                │
├────────────────┼────────────────────────────────────────────────────────┤
│ RobustKnock    │ 类似 OptKnock 但在生物质通量变化下保证鲁棒性           │
├────────────────┼────────────────────────────────────────────────────────┤
│ OptForce       │ 通过比较 WT 与期望状态，识别要过表达、低表达            │
│                │ 或删除的反应                                           │
├────────────────┼────────────────────────────────────────────────────────┤
│ FSEOF          │ 基于强制目标通量的通量扫描：扫描通量空间                │
│                │ 以找到与产物相关的反应                                  │
├────────────────┼────────────────────────────────────────────────────────┤
│ GECKO + MOMENT │ 使用 ecGEM 识别具有低 kcat 或低表达的                    │
│                │ 限速酶，它们是生产的瓶颈                                │
└────────────────┴────────────────────────────────────────────────────────┘
```

**OptKnock 背后的关键概念：**

```text
OptKnock 的双层问题：

  外层（工程师）：
    最大化：目标化合物的产量 (v_producto)
    选择：敲除哪些反应 (v_i = 0)

  内层（细胞）：
    最大化：生长速率 (v_biomasa)
    约束： S · v = 0,  lb ≤ v ≤ ub,  v_i = 0（对于被删除的基因）

  结果：一组敲除，使得当细胞试图最大化其自身生长时，
  不可避免地产生目标化合物作为强制性副产物。

  经典示例：大肠杆菌生产乳酸
    OptKnock 推荐：Δpta Δadhb Δppc
    → 重新氧化 NADH（生长所需）的唯一方式
      是通过乳酸脱氢酶 → 乳酸被强制生产
```

```text

  model = cobra.io.load_json_model("iJO1366.json")

步骤 3 — 定义培养条件
  → 根据实验培养基调整交换反应 (EX_)

  model.reactions.get_by_id("EX_o2_e").lower_bound = -20       # 氧气

  sol = model.optimize()
  print(sol.objective_value)   # 预测的生长速率
  # 与已发表的实验 μ 进行比较以验证

  model.objective = "EX_succ_e"   # 例如琥珀酸

步骤 6 — 探索工程靶点
  → 单/双敲除：single_gene_deletion(), double_gene_deletion()
  → Production envelope: production_envelope()
  → 部分下调：修改特定反应的 ub

步骤 7 — 解读和优先排序
  → 按预测产量排序
  → 筛选维持 μ > 最低阈值（例如 μ > 0.1 h⁻¹）的那些
  → 验证敲除在实验上是否可行

步骤 8 — 使用 ecGEM (GECKO) 验证（如果可用）
  → 从 github.com/SysBioChalmers/ecModels 加载对应的 ecGEM
  → 使用酶约束重复模拟
  → 比较 FBA vs. ecGEM 预测 → 丢弃被高估的候选
```

## 10. 概念总结

在本模块中你学到了：

- **代谢**可以表示为由酶促反应连接的代谢物的**网络（图）**，在结构上可与城市交通网络进行比较。
- **ODE** 允许模拟代谢物浓度在时间中如何变化，但需要难以在基因组规模上测量的动力学参数。
- **GEM 模型**通过假设**稳态**（S·v = 0）并将问题表述为线性优化来解决该局限性。
- **FBA** 在网络约束下找到最大化生物学目标（如生长）的最优通量分布。
- **GECKO** 通过纳入真实的酶能力（kcat, 蛋白质组预算）扩展 FBA，获得更逼真且对生物工厂设计更有用的预测。
- 整个流程**从基因组出发**：一个注释良好的基因组是任何代谢模型的初始输入。

> [!IMPORTANT]
> 代谢建模不是替代实验：而是指导实验。一个好的模型将实验搜索空间从数百种可能的遗传修饰缩小到少数几个成功概率更高的候选方案，加速代谢工程中的 DBTL 循环。
