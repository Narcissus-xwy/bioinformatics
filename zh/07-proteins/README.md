## 1. 蛋白质序列：从 DNA 到氨基酸

### 1.1 氨基酸：蛋白质的构建块

蛋白质是由 20 种标准氨基酸组成的线性聚合物。每个氨基酸由 mRNA 上的一个三联体密码子（codon）编码。

| 性质 | 氨基酸（单字母代码） |
|---|---|
| **疏水性（Hydrophobic）** | A, V, L, I, P, F, W, M |
| **极性（Polar）** | G, S, T, C, Y, N, Q |
| **带正电（Positive）** | K, R, H |
| **带负电（Negative）** | D, E |

> [!TIP]
> 氨基酸的性质决定了蛋白质如何折叠。疏水残基倾向于埋在蛋白质内部，而带电残基暴露在表面与水相互作用。

### 1.2 一级结构：序列

蛋白质的一级结构就是其氨基酸的线性序列，从 N 端（氨基端）到 C 端（羧基端）：

```text
N端 ─ Met ─ Ala ─ Lys ─ Leu ─ Gly ─ ... ─ Val ─ C端
```

在生物信息学中，蛋白质序列通常以单字母代码表示：

```text
>sp|P68871|HBB_HUMAN Hemoglobin subunit beta
MVHLTPEEKSAVTALWGKVNVDEVGGEALGRLLVVYPWTQRFFESFGDLSTPDAVMGNPKVKAHGKKVLG
AFSDGLAHLDNLKGTFATLSELHCDKLHVDPENFRLLGNVLVCVLAHHFGKEFTPPVQAAYQKVVAGVAN
ALAHKYH
```

### 1.3 关键蛋白质数据库

| 数据库 | 内容 | 链接 |
|---|---|---|
| **UniProt** | 全面的蛋白质序列和功能信息 | https://www.uniprot.org/ |
| **NCBI Protein** | NCBI 的蛋白质序列集合 | https://www.ncbi.nlm.nih.gov/protein/ |
| **PDB**（Protein Data Bank） | 实验确定的 3D 结构 | https://www.rcsb.org/ |
| **AlphaFold DB** | AI 预测的 3D 结构 | https://alphafold.ebi.ac.uk/ |

## 2. 结构域、基序和蛋白质家族

### 2.1 结构域（Domains）

**结构域**是蛋白质中可以独立折叠并通常具有特定功能的结构和功能单元。一个蛋白质可以包含一个或多个结构域。

```text
蛋白质架构示例（转录因子）：

  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐
  │ DNA 结合域   │──│ 连接区       │──│ 激活域       │
  │ (Pfam: PF0001)│  │              │  │ (Pfam: PF0002)│
  └──────────────┘  └──────────────┘  └──────────────┘
```

识别结构域有助于推断功能：如果一个未知蛋白质含有"激酶结构域"，它可能是一个激酶。

### 2.2 基序（Motifs）

**基序**是较短的保守序列模式，通常与特定功能相关：

| 基序 | 功能 |
|---|---|
| N-糖基化位点 | N-X-[S/T]（X ≠ P） |
| ATP 结合位点 | G-X-X-X-X-G-K-[T/S] |
| 锌指（Zinc finger） | C-X₂₋₄-C-X₁₂-H-X₃₋₅-H |

### 2.3 蛋白质家族数据库

| 数据库 | 描述 | 链接 |
|---|---|---|
| **Pfam** | 蛋白质家族和结构域 | https://www.ebi.ac.uk/interpro/ |
| **InterPro** | 蛋白质家族、结构域和功能位点的集成资源 | https://www.ebi.ac.uk/interpro/ |
| **CDD**（Conserved Domain Database） | NCBI 的保守结构域集合 | https://www.ncbi.nlm.nih.gov/cdd/ |
| **PROSITE** | 蛋白质结构域、家族和功能位点 | https://prosite.expasy.org/ |

## 3. 蛋白质搜索和注释

### 3.1 BLASTp：蛋白质对蛋白质搜索

当你有一条蛋白质序列并想知道它是什么时，BLASTp 是第一步：

```bash
# 示例：搜索血红蛋白 beta 亚基的同源物
# 在 NCBI BLAST (https://blast.ncbi.nlm.nih.gov/)：
#   - 选择 "Protein BLAST" (blastp)
#   - 粘贴蛋白质序列
#   - 选择数据库：nr（非冗余蛋白质）
```

**与 blastn 的关键区别：**
- blastp 使用替代矩阵（默认 BLOSUM62），而非简单的 match/mismatch
- 蛋白质搜索比核苷酸搜索更敏感，因为氨基酸字母表更大（20 vs 4）

### 3.2 解读 BLASTp 结果

| 参数 | 在蛋白质背景下的含义 |
|---|---|
| **E-value** | 在数据库中偶然找到该比对的概率 |
| **% Identity** | 比对中相同氨基酸的百分比 |
| **% Positives** | 相同或具有相似化学性质的氨基酸百分比 |
| **Query coverage** | query 序列被对齐的百分比 |

> [!WARNING]
> 在蛋白质中，低 identity（25-30%）并不意味着"不相关"。蛋白质可以在保持相同结构和功能的同时 sequence 上高度分化。这就是为什么"相似性（positives）"比纯粹的"同一性（identity）"更具信息量。

### 3.3 功能注释的工作流

```text
蛋白质序列（未知功能）
         │
         ▼
  ┌─────────────────┐
  │ 1. BLASTp vs    │  → 识别同源物
  │    UniProt/nr   │    （同一性 >30% 可能相关）
  └────────┬────────┘
           │
           ▼
  ┌─────────────────┐
  │ 2. 结构域搜索   │  → 识别功能模块
  │    Pfam/InterPro │    （结构域通常比全局序列更保守）
  └────────┬────────┘
           │
           ▼
  ┌─────────────────┐
  │ 3. 功能分配     │  → 整合证据
  │    GO terms     │    基因本体（Gene Ontology）术语：
  │    KEGG pathways│    分子功能、生物过程、细胞组分
  │    EC numbers   │
  └─────────────────┘
```

## 4. 蛋白质结构

### 4.1 从序列到结构

蛋白质结构有四个层次：

| 层次 | 描述 | 稳定作用力 |
|---|---|---|
| **一级** | 氨基酸序列 | 肽键 |
| **二级** | 局部折叠：α-螺旋、β-折叠 | 氢键 |
| **三级** | 整体 3D 折叠 | 疏水相互作用、二硫键、离子键 |
| **四级** | 多条多肽链的组装 | 相同的非共价相互作用 |

> [!NOTE]
> **Anfinsen 法则：** 蛋白质的氨基酸序列决定了它的三维结构。这被称为"蛋白质折叠问题"，AlphaFold 等 AI 工具正在帮助解决这个问题。

### 4.2 结构可视化

```text
蛋白质结构（以血红蛋白为例）：

     ╔═══  α-螺旋 (α-helix)
     ║
  ═══╬══════╗
     ║      ║  ← β-折叠 (β-sheet)
     ╚══════╣
            ║
 血红素 ═══╝  ← 辅因子（辅基）
     │
     │
    Fe²⁺  ← 结合 O₂
```

工具：
- **PyMOL**：专业分子可视化（需要许可证，但有教育版）
- **UCSF ChimeraX**：免费，功能强大，推荐用于学术用途
- **NGL Viewer**：基于 Web，内置于 PDB 网站

### 4.3 为什么结构重要？

| 应用 | 结构的作用 |
|---|---|
| **药物设计** | 设计能结合活性位点的小分子 |
| **突变效应预测** | 突变如何改变折叠或功能？ |
| **酶工程** | 修改活性位点以改变底物特异性 |
| **功能推断** | 在序列相似性不足时，结构相似性可能揭示功能 |
| **疾病机制** | 错义突变常导致错误折叠和聚集 |

> [!TIP]
> 在序列同一性很低（<20%）的情况下，蛋白质仍然可能共享相同的三维折叠。这种现象称为"结构比序列更保守"。

## 5. 在实践中：一个典型的蛋白质分析

假设你有一条来自新测序细菌基因组的未知蛋白质序列：

1. **BLASTp 搜索：** 找到 45% 同一性的匹配 → 可能是同一家族的成员。
2. **Pfam 分析：** 识别出一个"ATP 结合域"和一个"激酶结构域" → 支持激酶功能。
3. **InterPro 分析：** 分配 GO term "蛋白质磷酸化" → 特定的生物学功能。
4. **AlphaFold 预测：** 生成一个预测的 3D 结构，与已知激酶的结构匹配良好。
5. **结论：** 该蛋白质可能是一个 ATP 依赖的蛋白质激酶。

## 6. 概念总结

在本模块中你学到了：

- 蛋白质是氨基酸的线性聚合物，折叠成特定的三维结构；
- **结构域**是蛋白质的功能和结构模块，通过 Pfam 和 InterPro 等数据库进行注释；
- **BLASTp** 是鉴定蛋白质同源物的主要工具，使用替代矩阵而非简单的 match/mismatch；
- 蛋白质结构比序列**更保守**，即使序列同一性很低，结构相似性也可能揭示功能；
- 蛋白质功能注释是一个多步骤过程，结合同源性搜索、结构域分析和功能分配。

> [!IMPORTANT]
> 蛋白质生物信息学连接了基因组（我们测序和组装的内容）与功能（细胞实际做的事情）。理解蛋白质意味着将基因与生物学联系起来。
