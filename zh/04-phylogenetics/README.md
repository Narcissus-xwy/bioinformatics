# 模块 4：系统发育学与进化分析

## 引言

在模块 3 中，你学习了比较序列：BLAST、全局和局部比对、打分（scoring）、同一性（identity）vs. 相似性（similarity）。所有这些工具都允许你一次比较**两条**序列。但在生物学中，我们很少只处理两条序列。当你有一个来自 10 个不同细菌菌株的基因，或来自环境采样的 50 条 16S rRNA 序列时，你需要**同时**比较它们并理解它们之间的关系。

这正是系统发育学所做的：取一组序列，将它们对齐（align），估计它们的分化程度，并构建一个代表其进化关系的**系统发育树（phylogenetic tree）**。

本模块是前一个模块的自然延伸。如果说在模块 3 中你学会了比较序列的"方法"，那么在这里你将学会"目的"：重建连接生物体、基因或蛋白质的进化历史。

我们将围绕五个主要模块展开：

1. **基本概念** — 标记基因（marker genes）、数据库、为什么 BLAST 不足以鉴定物种以及 ANI 等替代方案。
2. **手工构建一棵树** — 在使用软件之前理解其逻辑。
3. **多序列比对（MSA, Multiple Sequence Alignment）** — 所有分子系统发育分析的基础。
4. **替代模型（Substitution Models）** — 如何测量序列之间真实的"进化距离"。
5. **构建系统发育树的方法** — 邻接法（Neighbor-Joining）、最大简约法（Maximum Parsimony）和最大似然法（Maximum Likelihood）。

> [!NOTE]
> 系统发育学不仅仅是学术练习。在临床微生物学中，它用于追踪医院暴发；在流行病学中，用于追踪病原体的进化（如 SARS-CoV-2 所做的那样）；在生物技术中，用于从相关生物中鉴定酶；在生态学中，用于研究微生物群落的多样性。

## 1. 基本概念：标记基因、鉴定及其局限性

在构建系统发育树之前，你需要理解**哪些序列被用于**系统发育学，更重要的是，相似性搜索**可以告诉你什么，不可以告诉你什么**关于一个生物的身份。

### 1.1 保守基因及其用途

并非基因组的所有区域都同样适用于研究进化关系。为了比较数百万年前分化的生物，我们需要这样的基因：

- 存在于目标群体中**所有**生物中（通用性）；
- 具有**保守**区域（以便进行比对）和**可变**区域（以便区分生物）交错排列；
- 以适当的速率进化：既不太快以至于变化饱和，也不太慢以至于不提供信息。

这些基因被称为**标记基因（marker genes）**或**系统发育标记（phylogenetic markers）**。最重要的标记基因来自**核糖体（ribosome）**。

> [!NOTE]
> 📌 核糖体：16S、23S、ITS 等名称的来源
>
> **核糖体**是将信使 RNA 翻译成蛋白质的细胞机器。所有生物都有核糖体，这使其成为通用标记的理想来源。它由两个大小不同的亚基组成，每个亚基由**核糖体 RNA（rRNA）**和蛋白质组成。
>
> "**S**"（如 16S, 23S）指的是**Svedberg**，一个测量粒子在超速离心机中沉降速度的单位。它不是直接的大小度量，而是分子在离心力场中"沉降"的方式——取决于大小、形状和密度。因此这些值不是加性的：一个 30S + 一个 50S 亚基不等于 80S，而是约 70S。
>
> * **原核生物核糖体（70S）**
> ```text
> 核糖体 70S（细菌和古菌）
> ├── 小亚基（30S）
> │   └── rRNA 16S  ← 微生物学中最常用的标记
> │
> └── 大亚基（50S）
>     ├── rRNA 23S  ← 辅助标记（更长，信息更多）
>     └── rRNA 5S   ← 非常短（约 120 nt），很少用作标记
> ```
>
> * **真核生物核糖体（80S）**
> ```text
> 核糖体 80S（真核生物：真菌、动物、植物、原生生物）
> ├── 小亚基（40S）
> │   └── rRNA 18S  ← 相当于原核生物的 16S
> │
> └── 大亚基（60S）
>     ├── rRNA 28S  ← 相当于 23S
>     ├── rRNA 5.8S ← 真核生物特有
>     └── rRNA 5S
> ```
>
> 在真核生物中，**ITS** 区域扮演关键角色：**ITS1**（位于 18S 和 5.8S 之间）和 **ITS2**（位于 5.8S 和 28S 之间）是**真菌**鉴定最常用的区域。为什么？
> - 核糖体基因（18S, 5.8S, 28S）非常保守 → 可以使用锚定在这些保守区域的**通用引物**来扩增 ITS。
> - ITS 区域在成熟核糖体中**没有功能**（被切除） → 处于较小的选择压力下 → 进化更快 → 在物种水平上具有更高的分辨率。

> [!TIP]
> 当你阅读一篇提到"16S 基因分析"的文章时，请记住它指的是编码**原核生物**核糖体小亚基 rRNA 的基因。如果生物是真菌，请寻找 ITS；如果是动物，寻找 COI；如果是植物，寻找 matK/rbcL。

#### 其他重要标记基因

| 标记 | 类群 | 特征 |
|---|---|---|
| **18S rRNA** | 真核生物 | 相当于真核生物的 16S；物种水平分辨率较低 |
| **rpoB** | 细菌 | RNA 聚合酶基因；当 16S 不能区分近缘物种时很有用 |
| **gyrB** | 细菌 | 促旋酶基因；良好的种内分辨率 |
| **recA** | 细菌 | 重组基因；常用于*伯克霍尔德菌属*（*Burkholderia*）等问题属 |
| **COI**（细胞色素 c 氧化酶 I） | 动物 | 动物的"DNA 条形码"；线粒体基因 |
| **matK / rbcL** | 植物 | 叶绿体基因；用作植物的 DNA 条形码 |

### 1.2 标记基因搜索的数据库

并非所有数据库都一样。在 BLAST 中选择正确的数据库直接影响结果的质量：

| 数据库 | 内容 | 使用时机 |
|---|---|---|
| **nr**（非冗余蛋白质） | 所有非冗余蛋白质序列 | 蛋白质同源搜索（`blastp`, `blastx`） |
| **nt**（核苷酸集合） | NCBI 中的所有核苷酸序列 | 通用核苷酸搜索（`blastn`） |
| **refseq_rna** | 经审编的参考 RNA（RefSeq） | 专门搜索转录本或经审编的核糖体 RNA |

#### 标记基因专用数据库

| 数据库 | 标记 | 链接 | 优势 |
|---|---|---|---|
| **SILVA** | 16S, 18S, 23S rRNA | https://www.arb-silva.de/ | 经审编，分类学对齐；16S 最常用 |
| **RDP**（Ribosomal Database Project） | 16S rRNA | https://rdp.cme.msu.edu/ | 带置信区间的分类学分类 |
| **Greengenes2** | 16S rRNA | https://greengenes2.ucsd.edu/ | 集成宏基因组学工具（QIIME 2） |
| **UNITE** | ITS（真菌） | https://unite.ut.ee/ | 真菌鉴定的标准参考 |
| **BOLD** | COI（动物） | https://www.boldsystems.org/ | 结合形态学数据的动物 DNA 条形码 |

### 1.3 为什么 16S rRNA 基因的 BLAST 不足以确定属和种？

这是微生物学和生物信息学中最常见的误解之一。

#### 典型场景

你对从临床或环境样本中分离的细菌的 16S rRNA 基因进行测序。你在 NCBI 的 `nt` 或 `16S ribosomal RNA` 数据库中做 BLAST。第一个结果显示：

```text
Klebsiella pneumoniae strain XYZ    99.5% identity    E-value: 0.0    Query coverage: 100%
```

立即的诱惑是得出结论：*"我的细菌是肺炎克雷伯菌（Klebsiella pneumoniae）"*。但这个结论**可能是错误的**，原因有以下几个：

#### 原因 1：16S rRNA 在物种水平上分辨率不足

16S 基因非常适合区分远缘的**科**和**属**，但许多近缘物种的 16S 序列**几乎相同**（>99% 同一性）。

充分记录的示例：

| 类群 | 问题 |
|---|---|
| *Escherichia coli* 和 *Shigella* spp. | 16S 同一性 >99%，但临床上非常不同 |
| *Bacillus cereus*, *B. anthracis*, *B. thuringiensis* | 16S 几乎无法区分（约 99.5–100%） |
| *Streptococcus pneumoniae* 和 *S. mitis* | 16S >99% 相同 |
| *Burkholderia* 复合群 | 许多物种的 16S 几乎相同 |

#### 原因 2：BLAST 搜索相似性，不鉴定

BLAST 告诉你你的序列与数据库中序列**有多相似**。它不告诉你你的生物**就是**那个物种：

- **数据库可能包含分类错误的序列**：GenBank 中有不可忽视比例的 16S 序列的分类学分配不正确。
- **最佳匹配（best hit）可能不是系统发育上最接近的**：BLAST 优化 score，而非系统发育。
- **同一性百分比取决于测序片段**：如果你只测了 V3–V4 区域（约 450 bp），该同一性仅针对该片段计算。

#### 原因 3：同一性阈值是指南，不是规则

| 16S 同一性 | 通常引用的解读 |
|---|---|
| ≥97% | 可能是同一物种 |
| ≥95% | 可能是同一属 |
| <95% | 可能为不同属 |

但这些阈值是几十年前建立的**统计近似值**。存在 16S 同一性 >99% 的不同物种，也存在 <97% 的种内变异。此外，97% 阈值是针对完整基因（约 1,500 bp）校准的；对于部分片段，数值不太可靠。

#### 那么 16S 的 BLAST 有什么用？

16S 的 BLAST **确实有用**，但作为**初步近似**，而非最终结论：

| 16S BLAST 可用于... | 16S BLAST 不能用于... |
|---|---|
| 将你的生物放置在广泛的分类学背景中（可能的科、属） | 确定物种 |
| 排除明显不同的候选（<95% 同一性） | 区分 16S 几乎相同的物种 |
| 为进一步详细分析确定生物类群 | 替代正式的系统发育分析 |

#### 那么该怎么做？

正确答案是：**构建一棵系统发育树**，这正是你在本模块中将要学习的。

### 1.4 超越 16S：ANI 和比较基因组学

对于 16S 分辨率不足的情况，现代解决方案是比较**完整基因组**：

#### ANI（Average Nucleotide Identity，平均核苷酸同一性）

**ANI** 计算两个完整基因组之间核苷酸同一性的平均百分比，仅考虑共享的同源区域。它目前是原核生物物种界定的**金标准**。

| ANI | 解读 |
|---|---|
| ≥95–96% | 同一物种 |
| 90–95% | 同一属，不同物种 |
| <90% | 不同属 |

工具：**JSpeciesWS**、**EzBioCloud ANI Calculator**、**fastANI**、**pyani**。

#### MLST（Multi-Locus Sequence Typing，多位点序列分型）

另一种替代方案是 **MLST**，它比较**多个持家基因**（通常 7 个）的片段，而非单个标记。每种独特的等位基因组合定义一个**序列型（ST, sequence type）**。MLST 比 16S 具有更好的分辨率，广泛用于医院流行病学。

> [!IMPORTANT]
> 一般规则是：**单个基因、单种方法、单个工具**几乎不足以在物种水平上可靠地进行分类学鉴定。系统发育学提供背景，ANI 提供精度，二者的结合提供可信度。BLAST 是起点，不是终点。

## 2. 在序列之前：手工构建一棵树

在与 DNA 序列和专业软件打交道之前，理解**系统发育树背后的逻辑**至关重要。而理解它的最佳方式就是亲手构建一棵。

### 2.1 什么是系统发育树？

**系统发育树**是表示一组生物（或基因、蛋白质）之间进化亲缘关系的图表。分支分开的点（**节点**）代表共同祖先，分支的长度可以代表进化变化的量或经过的时间。

树的基本组成部分：

```text
         ┌── 物种 A
    ┌────┤
    │    └── 物种 B
────┤
    │    ┌── 物种 C
    └────┤
         └── 物种 D
```

| 元素 | 代表什么 |
|---|---|
| **叶节点（leaf）** | 被比较的现有生物 |
| **内部节点（internal node）** | 假设的共同祖先 |
| **分支（branch）** | 进化谱系；长度可表示分化程度 |
| **根（root）** | 整群最古老的共同祖先（如果树有根） |
| **支（clade）** | 一个祖先及其所有后代（一个"自然群"） |

### 2.2 实践示例：使用形态学特征构建树

让我们在不使用任何软件或 DNA 序列的情况下构建一棵系统发育树。我们只需要一张**特征**表（性状的有/无）和逻辑。

#### 数据

我们有五种脊椎动物和五个形态学特征。**七鳃鳗（Lamprey）**作为**外群（outgroup）**：

| 特征 | 七鳃鳗 | 鲈鱼 | 凯门鳄 | 秃鹰 | 羚羊 |
|---|---|---|---|---|---|
| 下颌 | — | ✓ | ✓ | ✓ | ✓ |
| 肺 | — | — | ✓ | ✓ | ✓ |
| 砂囊 | — | — | ✓ | ✓ | — |
| 羽毛 | — | — | — | ✓ | — |
| 毛发 | — | — | — | — | ✓ |

#### 步骤 1：识别外群

**七鳃鳗**是唯一没有下颌的生物。由于所有其他生物都有下颌，七鳃鳗代表了最祖先的状态。这使其成为**外群（outgroup）**。

#### 步骤 2：按共享特征分组

现在我们寻找允许对生物进行分组的**共享衍生特征（synapomorphy）**：

1. **下颌** → 存在于鲈鱼、凯门鳄、秃鹰和羚羊，但不在七鳃鳗 → 将这四种归为一个支（**有颌类**）。
2. **肺** → 存在于凯门鳄、秃鹰和羚羊，但不在鲈鱼 → 将这三种归为一个亚群（**四足类**）。
3. **砂囊** → 存在于凯门鳄和秃鹰，但不在羚羊 → 将凯门鳄和秃鹰归为更近的关系（**主龙类**）。
4. **羽毛** → 仅存在于秃鹰 → **自有衍征（autapomorphy）**，不分组。
5. **毛发** → 仅存在于羚羊 → 另一个自有衍征。

#### 步骤 3：构建树

```text
                            ┌─── 秃鹰        [下颌, 肺, 砂囊, 羽毛]
                     ┌──────┤
                     │      └─── 凯门鳄      [下颌, 肺, 砂囊]
              ┌──────┤
              │      └────────── 羚羊         [下颌, 肺, 毛发]
       ┌──────┤
       │      └──────────────── 鲈鱼          [下颌]
───────┤
       └────────────────────── 七鳃鳗         [— (外群)]
```

> [!IMPORTANT]
> 这与处理 DNA 序列时使用的逻辑完全相同，只是将"有下颌/无下颌"替换为"在位置 150 有 A/在位置 150 有 G"。特征变了，但推理不变。

### 2.4 从纸到 DNA

你刚刚手工做的在概念上与使用软件所做的完全相同：

| 手工（本练习） | 使用软件（模块其余部分） |
|---|---|
| 形态学特征表 | 多序列比对（MSA） |
| 特征的有/无 | 每个位置的核苷酸（A, T, G, C） |
| 按共享特征分组 | 计算距离或评估模型 |
| 在纸上画树 | 使用 NJ、ML 或简约法构建树 |

区别在于，使用 DNA 序列时你有**成百上千个"特征"**（比对位置），这使得手工操作变得不可能，但在统计上更加稳健。

## 3. 多序列比对（MSA）

### 3.1 什么是 MSA 以及为什么需要它？

在模块 3 中你学会了比对**两条序列**（两两比对，pairwise alignment）。**多序列比对**（*Multiple Sequence Alignment*, MSA）将其扩展到**同时三条或更多序列**。

MSA 是任何分子系统发育分析中**必须的第一步**。没有好的比对，接下来的一切——距离、模型、树——都将被扭曲。

```text
序列 1:   A T G C G A - T C G
序列 2:   A T G C - A A T C G
序列 3:   A T G C G A A T C A
序列 4:   A T G C G A - T C G
         * * * *     * * *
```

MSA 的每**列**代表一个同源位置（homologous position）。保守列（标有 `*`）对确认同源性信息量最大；可变列对区分生物信息量最大。

### 3.3 如何构建 MSA？

为超过几条序列构建最优 MSA 在计算上是不可能的（复杂度呈指数增长）。因此使用**启发式方法**，最常见的是**渐进式**方法：

#### 渐进式策略（Clustal, MUSCLE, MAFFT）

1. **比对所有序列对**并计算每对之间的相似性分数。
2. **构建引导树**（*guide tree*）指示哪些序列更相似。
3. **渐进式比对**：先比对最相似的序列，然后按引导树的顺序逐步纳入更远的序列。

> [!WARNING]
> 引导树**不是**最终的系统发育树。它只是一个决定比对顺序的工具。

### 3.4 MSA 工具

| 工具 | 特征 | 链接 |
|---|---|---|
| **ClustalW / Clustal Omega** | 经典方法；Omega 是现代版本 | https://www.ebi.ac.uk/jdispatcher/msa/clustalo |
| **MUSCLE** | 快速且精确；良好的通用选择 | https://www.ebi.ac.uk/jdispatcher/msa/muscle |
| **MAFFT** | 非常灵活；适合大型集合或高度分化的序列 | https://mafft.cbrc.jp/alignment/server/ |
| **MEGA** | 集成 ClustalW 和 MUSCLE；适合完整流程（MSA → 树） | https://www.megasoftware.net/ |

## 4. 替代模型

### 4.1 为什么不能只统计差异？

如果两条序列在 100 个位置中有 10 个不同，我们可以说有 10% 的差异。但这低估了真实距离，因为：

- 一个位置可能**改变过不止一次**（例如 A → G → A，看起来"无变化"但实际发生了两次替代）；
- 并非所有替代发生的概率相同（**转换 transition** A↔G, C↔T 比**颠换 transversion** A↔C, A↔T, G↔C, G↔T 更频繁）；
- 每种核苷酸在基因组中的频率因生物而异。

**替代模型（substitution models）**纠正这些偏倚并估计序列之间"真实"的进化距离。

### 4.3 常见模型（从简单到复杂）

| 模型 | 参数 | 主要假设 |
|---|---|---|
| **JC69**（Jukes-Cantor） | 1 | 所有替代的概率相同；碱基频率相等（各 25%） |
| **K2P**（Kimura 2-parameter） | 2 | 区分转换和颠换；频率相等 |
| **HKY85**（Hasegawa-Kishino-Yano） | 5 | 区分转换和颠换 + 允许碱基频率不等 |
| **GTR**（General Time Reversible） | 9 | 每种替代类型都有其自身的速率；自由频率。最通用的模型 |

> [!TIP]
> 理解每个模型对**DNA 如何进化**做出假设比记住公式更重要。更复杂的模型并不总是更好——如果序列较少，可能会过拟合。

## 5. 构建系统发育树的方法

### 5.1 总体分类

| 家族 | 主要方法 | 工作原理 |
|---|---|---|
| **基于距离** | Neighbor-Joining (NJ), UPGMA | 计算所有序列之间的距离矩阵，然后分组 |
| **基于特征** | Maximum Parsimony (MP), Maximum Likelihood (ML) | 直接评估比对位置以推断树 |

### 5.2 邻接法（Neighbor-Joining, NJ）

最常用的基于距离的方法。

- **优势：** 非常快，适合大型集合，良好的起点
- **局限性：** 只产生**一棵**树，信息在压缩为距离时会丢失细节

### 5.3 最大简约法（Maximum Parsimony, MP）

简约原则说：**最佳树是解释数据所需进化变化最少的树**。

- **优势：** 直观，不需要替代模型
- **局限性：** 在进化速率非常不均衡时可能不一致（*长枝吸引 long-branch attraction*）

### 5.4 最大似然法（Maximum Likelihood, ML）

当今最稳健且最广泛使用的方法。

- **优势：** 统计上严格，结合显式替代模型，对*长枝吸引*等伪影稳健
- **局限性：** 计算密集，需要选择合适的模型

### 5.5 方法快速比较

| 标准 | NJ | MP | ML |
|---|---|---|---|
| **速度** | ⚡⚡⚡ 非常快 | ⚡ 慢 | ⚡⚡ 中等 |
| **替代模型** | 是（用于距离） | 否 | 是（显式） |
| **最优性标准** | 最小总长度 | 最少变化数 | 最大概率 |
| **推荐用途** | 快速探索，大型集合 | 形态学数据，近缘序列 | 最终分析，发表 |

### 5.6 如何知道树是否可靠？— Bootstrap

一棵系统发育树是一个**假设**，不是确定性事实。为了评估树的每个分支有多可靠，使用 **bootstrap**：

1. 对对齐进行**有放回重采样（re-sample）**。
2. 用重采样的对齐构建一棵树。
3. 重复此过程**成百上千次**（通常 100–1000 次）。
4. 对于原始树的每个分支，计算该相同分组在复制品中出现的**百分比**。

| Bootstrap 值 | 解读 |
|---|---|
| ≥ 95% | 非常强的支持 |
| 70–95% | 中等至良好的支持 |
| 50–70% | 弱支持 |
| < 50% | 该分支不可靠 |

## 6. 完整工作流

```text
序列 (FASTA)
       │
       ▼
多序列比对 (MSA)                  ← ClustalW, MUSCLE, MAFFT
       │
       ▼
检查和修剪                         ← AliView, trimAl
       │
       ▼

       │
       ▼

       │
       ▼
评估 (Bootstrap)                  ← 100–1000 次重复
       │
       ▼

```

## 7. 概念总结

在本模块中你学到了：

- 手工使用**形态学特征**构建树，以理解软件背后的逻辑；
- **MSA** 是所有系统发育分析的基础，其质量决定结果质量；
- **替代模型**纠正测量进化距离时的偏倚；
- 存在多种**构建树的方法**，每种都有其优势和局限性；
- **bootstrap** 是评估每个分支置信度的标准工具。

> [!IMPORTANT]
> 一棵系统发育树是一个假设，不是事实。其强度取决于数据质量、模型的适当性以及与其他类型证据的一致性。
