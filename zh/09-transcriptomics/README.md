# 模块 9：转录组学（RNA-Seq）

## 引言

如果说基因组是固定的（在所有细胞中都相同），**转录组**则是动态的：它反映了在任何给定时刻哪些基因是活跃的。转录组学——研究 RNA 转录本的全套集合——是理解基因调控、疾病机制和细胞身份的最强大工具之一。

本模块涵盖 RNA-Seq 分析的实验基础和生物信息学工作流：

1. **RNA-Seq 是什么** — 实验设计和偏差
2. **从读长到计数** — 比对和定量
3. **差异表达分析** — 在条件之间比较
4. **功能解读** — 给基因列表赋予意义

> [!NOTE]
> RNA-Seq 让我们能够"监听"细胞在做什么。当一个基因被转录时，意味着细胞需要它的产物。通过测量所有转录本，我们可以推断细胞的生理状态。

## 1. 什么是 RNA-Seq？

### 1.1 基本原理

RNA-Seq 使用高通量测序来定量 RNA 转录本。通用工作流为：

```text
        ┌─────────────────────┐
        │ 1. 提取 RNA        │
        │    （总 RNA 或      │
        │     mRNA 富集）     │
        └──────────┬──────────┘
                   │
                   ▼
        ┌─────────────────────┐
        │ 2. 文库制备        │
        │    将 RNA 转化为    │
        │    cDNA             │
        └──────────┬──────────┘
                   │
                   ▼
        ┌─────────────────────┐
        │ 3. 测序            │
        │    获得读长 (FASTQ) │
        └──────────┬──────────┘
                   │
                   ▼
        ┌─────────────────────┐
        │ 4. 生物信息学      │
        │    比对 → 计数 →    │
        │    差异表达         │
        └─────────────────────┘
```

### 1.2 实验设计中的关键概念

| 概念 | 描述 | 为什么重要 |
|---|---|---|
| **生物学重复（Biological replicates）** | 来自独立生物样本的多个样本 | RNA-Seq 的首要要求；至少 3 个 |
| **技术重复（Technical replicates）** | 同一生物样本的多次测量 | 有用但不是必需的 |
| **批量效应（Batch effect）** | 由不同日期或技术人员处理样本导致的非生物学变异 | 如果处理不当可能掩盖生物信号 |
| **配对末端（Paired-end）** | 从两端读取 | 改善比对，尤其在重复区域 |
| **链特异性（Strand-specific）** | 保留转录方向的文库 | 区分重叠的正义和反义转录本 |

> [!WARNING]
> **没有生物学重复的 RNA-Seq 不能用于统计推断。** 这是不可协商的。没有重复，你无法估计生物学变异性，也无法计算 p-values。

### 1.3 RNA-Seq 数据的偏差

| 偏差 | 来源 | 缓解方法 |
|---|---|---|
| **GC 含量** | 文库制备中 GC 含量对 PCR 效率的影响 | 归一化方法，选择替代比对算法 |
| **基因长度** | 较长的基因产生更多读长 | 归一化为 TPM 或 FPKM |
| **rRNA 污染** | 核糖体 RNA 占总 RNA 的 >80% | 使用 poly-A 选择或 rRNA 去除 |
| **链偏差** | 如果不使用链特异性文库可能丢失转录方向信息 | 在比对器中设置 `--stranded` 标志 |

## 2. 从读长到计数

### 2.1 比对策略

对于 RNA-Seq，比对必须处理**剪接读长（spliced reads）**——跨越内含子的读长：

```text
基因结构：     外显子1   外显子2   外显子3
              ═══════╗  ╔══════╗  ╔══════
                     ║  ║      ║  ║
                     ╚══╝      ╚══╝
                 内含子1   内含子2

剪接读长：
   ═══════╗           ╔══════
          ╚═══════════╝
        （跨越内含子1）
```

| 比对器 | 类型 | 何时使用 |
|---|---|---|
| **STAR** | 剪接感知（Splice-aware） | 真核生物 RNA-Seq 的标准；快速且灵敏 |
| **HISAT2** | 剪接感知 | STSR 的替代方案，内存使用较少 |
| **Bowtie2** | 非剪接感知 | 原核生物 RNA-Seq（无剪接） |
| **Kallisto / Salmon** | 伪比对（Pseudoalignment） | 基于 k-mer 的快速定量，无需完整 BAM 文件 |

### 2.2 定量：从比对到计数

比对之后，你需要统计每个基因有多少条读长（**计数，count**）：

```text
基因 A:   ════════════════════
          读长: ▬▬▬▬  ▬▬▬  ▬▬▬▬

计数 = 3 条读长映射到基因 A

基因 B:   ════════════
          读长: ▬▬▬▬▬▬▬▬▬▬▬▬

计数 = 12 条读长映射到基因 B
```

**计数矩阵（count matrix）**是 RNA-Seq 分析的核心：

```text
           Sample_1  Sample_2  Sample_3  Sample_4
Gene_A        150      180      120      200
Gene_B       1200     1100     1300     1150
Gene_C         30       25       40       35
...
```

工具：**featureCounts**（来自 subread 包）、**HTSeq-count**。

### 2.3 归一化

原始计数不能直接比较，因为它们受测序深度和基因长度的影响：

| 指标 | 公式 | 何时使用 |
|---|---|---|
| **RPKM / FPKM** | 计数 / (基因长度 × 总读长数) × 10⁹ | 比较单个样本内的基因 |
| **TPM** | （RPKM / Σ RPKM）× 10⁶ | 比较样本之间（优于 RPKM） |
| **CPM** | 计数 / 总读长数 × 10⁶ | 简单的深度归一化 |

> [!TIP]
> 对于差异表达分析，大多数软件包（DESeq2, edgeR）优先使用**原始计数**并在内部应用其自身的归一化。不要预先归一化后再输入 DESeq2。

## 3. 差异表达分析

### 3.1 什么是差异表达？

一个基因如果在两种条件之间（例如，处理 vs 对照）以统计显著的方式改变其表达水平，则该基因是**差异表达（differentially expressed）**的。

```text
                    ┌──────────────────┐
                    │ 条件 A（对照）   │
                    └────────┬─────────┘
                             │
                             ▼
        计数矩阵 ──▶ 统计检验 ──▶ DEG 列表
                             ▲
                             │
                    ┌────────┴─────────┐
                    │ 条件 B（处理）   │
                    └──────────────────┘
```

### 3.2 统计方法

| 工具 | 方法 | 注意事项 |
|---|---|---|
| **DESeq2** | 负二项分布，使用经验贝叶斯收缩 | 最流行；能很好地处理小样本 |
| **edgeR** | 负二项分布，使用精确检验 | 快速；适合实验设计 |
| **limma-voom** | 使用 precision weights 的线性模型 | 灵活；能处理复杂设计 |

### 3.3 解读结果

差异表达分析的典型输出：

```text
Gene      log2FC   p-value    padj
Gene_A     2.5     0.0001    0.002
Gene_B    -1.8     0.0005    0.008
Gene_C     0.3     0.1000    0.500
...
```

| 列 | 含义 |
|---|---|
| **log2FC** | 对数 2 倍数变化（log2 Fold Change）。正值 → 在处理中上调；负值 → 下调 |
| **p-value** | 观察到的差异由随机性造成的概率 |
| **padj**（校正 p-value） | 针对多重检验校正的 p-value（通常是 Benjamini-Hochberg） |

> [!IMPORTANT]
> 始终使用**校正后的 p-value（padj）**。在测试 20,000 个基因时，1000 个会仅凭偶然就具有 p < 0.05。FDR 校正解释了这一点。

### 3.4 可视化

| 图 | 显示什么 |
|---|---|
| **火山图（Volcano Plot）** | log2FC vs -log10(padj)。显著基因像"火山"一样出现在顶部 |
| **MA 图** | log2FC vs 平均表达。显示低表达基因中是否偏差更大 |
| **热图（Heatmap）** | 在样本中聚类差异表达基因。显示表达模式 |
| **PCA 图** | 样本在主成分空间中的位置。检查样本是否按条件聚类 |

## 4. 功能解读

### 4.1 从基因列表到生物学

获得差异表达基因（DEG）列表后，真正的工作开始于：**这些基因意味着什么？**

```text
差异表达基因列表（例如，500 个基因）
         │
         ▼
  ┌──────────────┐
  │ 过表达分析   │  ← 相对于背景，哪些类别被富集？
  │ （ORA）      │    GO, KEGG, Reactome, Hallmark
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │ GSEA         │  ← 基因集富集分析：
  │              │    使用所有基因的排名列表，
  │              │    而非仅使用 DEG
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │ 生物学叙事   │  ← "处理 X 激活了 Y 通路，
  │              │    导致 Z 表型"
  └──────────────┘
```

### 4.2 工具

| 工具 | 功能 | 链接 |
|---|---|---|
| **clusterProfiler**（R） | GO 和 KEGG 的过表达和 GSEA | Bioconductor |
| **g:Profiler** | GO、通路和调控基序富集 | https://biit.cs.ut.ee/gprofiler/ |
| **Enrichr** | 多数据库富集分析 | https://maayanlab.cloud/Enrichr/ |
| **DAVID** | 经典工具，简单但具有有限的类别 | https://david.ncifcrf.gov/ |

## 5. 原核生物 vs 真核生物 RNA-Seq

| 方面 | 原核生物 | 真核生物 |
|---|---|---|
| **mRNA 富集** | 无 poly-A 尾 → 必须先去除 rRNA | poly-A 选择简单直接 |
| **剪接** | 很少（自剪接内含子除外） | 普遍存在；需要剪接感知比对器 |
| **读长计数** | 映射整个基因 | 按外显子计数，然后按基因汇总 |
| **操作子** | 多顺反子转录本（polycistronic） | 主要是单顺反子转录本（monocistronic） |
| **链特异性** | 推荐以避免操作子重叠 | 推荐用于区分正义和反义转录本 |

> [!TIP]
> 对于原核生物 RNA-Seq，**链特异性（stranded）文库**尤为重要，因为基因在两条链上密集排列且经常重叠。

## 6. 概念总结

在本模块中你学到了：

- RNA-Seq 通过 cDNA 测序**定量基因表达**；
- 实验设计至关重要：**生物学重复**是强制性的，**批量效应**必须被控制；
- 比对必须是**剪接感知的**（对于真核生物），并且需要为多顺反子转录本（原核生物）定制策略；
- **差异表达**使用负二项模型（DESeq2, edgeR）进行统计检验，始终使用**校正的 p-values**；
- 功能解读将基因列表转化为**生物学叙事**：哪些通路被激活或抑制？

> [!IMPORTANT]
> RNA-Seq 将你从"基因是什么？"（基因组学）带到"基因在做什么？"（转录组学）。这是理解基因调控和细胞状态最直接的方法。
