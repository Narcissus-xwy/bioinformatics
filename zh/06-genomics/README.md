# 模块 6：基因组学 — 从组装基因组到生物学知识

## 引言

在模块 5 中，你学习了如何生成测序读长、如何评估其质量以及如何通过组装重建基因组。该过程的输出是一个包含 contig 或 scaffold 的 FASTA 文件：一条长序列，但仍然是"沉默的"——我们不知道基因在哪里、它们执行什么功能，以及它们与其他生物相比有什么不同。

本模块的重点是为组装好的基因组赋予**生物学意义**。我们将围绕三个主要模块展开：

1. **基因组注释（Genome Annotation）** — 鉴定基因、分配功能并生成可解读的文件（GFF, GBK）。
2. **基因组变异（Genomic Variants）** — 通过与参考序列比对检测 SNP 和 indel。
3. **比较基因组学（Comparative Genomics）** — 比较完整基因组：ANI、泛基因组（pan-genome）、共线性（synteny）和基因组岛（genomic islands）。

> [!NOTE]
> 在基因组学中，获得一条长序列还不够；还必须证明它足够完整、一致且在生物学上可解读。本模块教你如何从序列走向知识。

## 1. 基因组注释

一旦你有了 FASTA 格式的组装结果，下一步是将这段"原始"序列转化为一个**可解读的生物学地图**：基因在哪里？它们做什么？

### 1.1 什么是基因？

从生物学角度来看，基因是产生功能性产物（RNA 或蛋白质）的 DNA 区域。从计算角度来看，注释旨在识别指示该功能单元起始和终止位置的**序列模式**。

### 1.2 结构注释 vs. 功能注释

#### 结构注释（Structural Annotation）

旨在识别**有哪些元素以及它们在哪里**：

- 编码基因（CDS, Coding Sequence）；
- tRNA 和 rRNA；
- 调控区域（启动子、终止子）；
- 真核生物中的外显子和内含子。

#### 功能注释（Functional Annotation）

旨在通过将序列与已知数据库和模型进行比较，推断每个基因或基因产物**做什么**。

### 1.3 如何识别基因？

预测算法寻找统计和生物学信号：

```text
典型原核生物基因：

  启动子   RBS   起始密码子   ──── 开放阅读框（ORF） ────   终止密码子
    ─┤      ┤     ATG ─────────────────────────────────── TAA/TAG/TGA
                     ↑                                              ↑
               注释程序寻找具有密码子偏好性的长 ORF
                  以及上游的调控信号
```

程序寻找的信号：

- **起始和终止密码子**（ATG → TAA/TAG/TGA）；
- 足够长的**开放阅读框（ORF, Open Reading Frame）**；
- **密码子使用偏好性**（每种生物偏好使用某些同义密码子）；
- **启动子信号**和核糖体结合位点（RBS）；
- 真核生物中的**剪接位点（splicing sites）**；
- 与已知基因的**相似性**（BLAST, Pfam 模型）。

### 1.4 原核生物 vs. 真核生物

| 特征 | 原核生物 | 真核生物 |
|---|---|---|
| 基因组织 | 紧凑，基因连续 | 分散，基因被中断 |
| 内含子 | 通常不存在 | 常见 |
| 基因密度 | 高（约 85% 编码） | 低（人类约 1.5%） |
| 计算预测 | 相对简单 | 复杂得多 |
| 有用的额外证据 | 同源性、保守基序 | RNA-Seq、同源蛋白质、复杂模型 |

```text
原核生物基因：                    真核生物基因：
                                          Exón  Exón  Exón  Exón
                                           ↑      ↑     ↑     ↑
ATG═══════════════TGA                 ATG══╗  ╔══╗  ╔══╗  ╔═══TGA
    ↑ ORF 连续 ↑                          └──┘  └──┘  └──┘
                                              ↑     ↑     ↑
                                           Intrón Intrón Intrón
                                     
                                    成熟 mRNA 仅包含外显子
```

#### 原核生物工具

| 工具 | 功能 | 备注 |
|---|---|---|
| **Prodigal** | 基因预测（CDS） | 快速，对原核生物非常精确 |
| **Prokka** | 完整流程：预测 + 功能注释 | 细菌基因组学的标准工具 |
| **Bakta** | Prokka 的现代替代方案 | 数据库更新更及时 |

#### 真核生物工具

| 工具 | 功能 | 备注 |
|---|---|---|
| **Augustus** | 使用训练模型的从头（*ab initio*）预测 | 需要按物种提供的训练数据 |
| **MAKER** | 结合 RNA-Seq 和蛋白质证据的完整流程 | 更复杂但更精确 |

### 1.5 功能注释

预测出基因后，推断其功能：

```text
预测的基因（氨基酸序列）
         │
         ▼
  ┌─────────────┐     ┌───────────────┐     ┌─────────────────┐
  │  BLAST vs   │     │  结构域搜索   │     │  类别分配       │
  │  UniProt/nr │     │  Pfam/InterPro│     │  GO, KEGG, COG   │
  └──────┬──────┘     └──────┬────────┘     └────────┬────────┘
         │                   │                       │
         ▼                   ▼                       ▼
    "蛋白酶"          "丝氨酸蛋白酶结构域"    "类别：代谢"
```

常用策略：

- 使用 BLAST 进行**同源性搜索**，比对 NCBI nr 或 UniProt 等数据库；
- 使用 Pfam、InterProScan 或 CDD 进行**结构域检测**；
- 使用 GO（Gene Ontology，基因本体论）、KEGG（代谢通路）或 COG 进行**功能分配**。

> [!WARNING]
> 功能注释通常是基于相似性的**预测**，而非直接的实验证明。一个与蛋白酶具有 40% 同一性的基因*可能*是蛋白酶，但也可能已经在功能上发生了分化。

### 1.6 输出格式：GFF 和 GBK

**GFF** 和 **GBK**（GenBank）格式用于在基因组序列上表示注释信息。简而言之：

- **FASTA** → "序列是什么？"
- **GFF** → "特征在哪里？"（坐标、类型、链、属性）
- **GBK** → 序列 + 注释集成在一个文件中

> [!TIP]
> 如需复习这些格式的结构和字段 → [模块 1 → 文件格式](../01-introduction/README.md)。

## 2. 基因组变异：SNP 和 indel

除了注释基因组，很多时候还需要将样本与参考序列进行比较，以识别点突变。

### 2.1 什么是变异？

**基因组变异（genomic variant）**是相对于另一条参考序列的序列差异：

```text
参考序列：  A T G C C G T A
样本：      A T G T C G T A
                 ↑
                SNP (C → T)

参考序列：  A T G C C G T A
样本：      A T G C - G T A
                   ↑
               缺失 (C 被删除)

参考序列：  A T G C C G T A
样本：      A T G C A C G T A
                   ↑
              插入 (A 被添加)
```

| 类型 | 定义 |
|---|---|
| **SNP**（Single Nucleotide Polymorphism） | 单个碱基的改变 |
| **Indel** | 短的插入或缺失 |

### 2.2 如何检测？

```text
变异检测流程：

      读长 (FASTQ)
              │
              ▼
   比对到参考序列                   ← BWA, Bowtie2, Minimap2
              │
              ▼
  比对文件 (BAM)
              │
              ▼
     变异检出（variant calling）    ← bcftools, GATK, Snippy
              │
              ▼
  变异文件 (VCF)
              │
              ▼
    筛选和注释                      ← SnpEff, SnpSift
```

#### VCF 格式（Variant Call Format）

这是报告变异的标准格式：

```text
#CHROM  POS   ID  REF  ALT  QUAL  FILTER  INFO
contig1 1045  .   C    T    200   PASS    DP=45;MQ=60
contig1 2300  .   AT   A    150   PASS    DP=38;MQ=55
```

| 字段 | 含义 |
|---|---|
| CHROM | 染色体或 contig |
| POS | 变异位置 |
| REF | 参考序列中的碱基 |
| ALT | 样本中的碱基 |
| QUAL | 检出质量 |
| DP | 该位置的测序深度 |

### 2.3 变异检出工具

| 工具 | 适用场景 | 备注 |
|---|---|---|
| **Snippy** | 细菌基因组学 | 完整流程：mapping + variant calling |
| **bcftools** | 通用 | 灵活，属于 samtools 的一部分 |
| **GATK** | 人类/真核基因组学 | 非常稳健，临床标准工具 |
| **FreeBayes** | 通用 | 贝叶斯方法，适合单倍型分析 |

### 2.4 变异的重要性

| 应用 | 示例 |
|---|---|
| 区分近缘菌株 | 通过 SNP 追踪医院暴发 |
| 抗微生物耐药性 | *gyrA* 基因突变 → 氟喹诺酮耐药 |
| 进化和精细系统发育 | 重建 SARS-CoV-2 在患者间的传播 |
| 毒力 | 鉴定毒力因子中的突变 |

### 2.5 解读变异的注意事项

并非所有观察到的差异在生物学上都是真实的。假阳性来源包括：

- 该位置读长质量低；
- 覆盖度不足（<10×）；
- 歧义比对（重复区域）；
- 所选参考序列引入的偏倚。

> [!TIP]
> 始终按质量（QUAL）、深度（DP）和比对质量（MQ）筛选变异。一个在覆盖度 5× 的区域仅由 3 条读长支持的变异是不可靠的。

## 3. 比较基因组学

比较基因组学旨在理解完整基因组之间的差异和相似性。它超越了单基因比较：这里同时比较**数千个基因**。

### 3.1 ANI（Average Nucleotide Identity）

我们已在[模块 4](../04-phylogenetics/README.md)中将其作为 16S 的替代方案用于物种界定。ANI 计算两个基因组之间同源区域核苷酸同一性的平均百分比：

| ANI | 解读 |
|---|---|
| ≥95–96% | 同一物种 |
| 90–95% | 同一属，不同物种 |
| <90% | 不同属 |

工具：**fastANI**、**pyani**、**JSpeciesWS**、**EzBioCloud ANI Calculator**。

### 3.2 泛基因组（Pan-genome）

当比较同一物种的多个基因组时，可以观察到并非所有基因都存在于所有分离株中。**泛基因组**描述了一个物种的完整基因库：

```text
一个细菌物种的泛基因组：

┌─────────────────────────────────────────────────────────┐
│                                                         │
│    ┌───────────────────┐                                │
│    │   核心基因组       │  ← 存在于所有分离株中的基因    │
│    │   (Core genome)    │    （必需基因）                │
│    └───────────────────┘                                │
│                                                         │
│    ┌──────────┐ ┌──────────┐ ┌──────────┐               │
│    │ 附属基因 │ │ 附属基因 │ │ 附属基因 │               │
│    │ 菌株 A   │ │ 菌株 B   │ │ 菌株 C   │  ← 可变基因  │
│    └──────────┘ └──────────┘ └──────────┘               │
│                                                         │
│    ┌─────┐  ← 特有基因（singleton）：                  │
│    │特有  │    仅存在于一个菌株中                       │
│    └─────┘                                              │
└─────────────────────────────────────────────────────────┘
```

| 组成 | 定义 | 生物学示例 |
|---|---|---|
| **核心基因组（Core genome）** | 存在于 ≥95% 分离株中的基因 | 持家基因（housekeeping）、核糖体基因 |
| **附属基因组（Accessory genome）** | 存在于部分分离株中的基因 | 致病岛、质粒 |
| **特有基因（Singleton）** | 仅存在于一个分离株中的基因 | 前噬菌体、近期移动元件 |

工具：**Roary**、**Panaroo**、**PIRATE**。

### 3.3 抗性和毒力基因检测

基因组学的一个直接应用是在组装好的基因组中搜索具有临床意义的基因：

| 搜索类型 | 数据库 | 工具 |
|---|---|---|
| **抗微生物耐药性（AMR）** | CARD, ResFinder, AMRFinderPlus | ABRicate, AMRFinderPlus, RGI |
| **毒力因子** | VFDB | ABRicate, VFanalyzer |
| **质粒** | PlasmidFinder | MOB-suite, PlasmidFinder |
| **分型（MLST）** | PubMLST | mlst (Seemann), MLST 2.0 |

> [!NOTE]
> 这些搜索针对经过审编（curated）的数据库进行，内部通常使用 BLAST 或 k-mer 比较。结果取决于数据库和组装的质量。

### 3.4 共线性（Synteny）和重排

**共线性**指的是基因在基因组之间顺序和方向的保守性：

```text
基因组 A:  ═══ gen1 → gen2 → gen3 → gen4 → gen5 ═══

基因组 B:  ═══ gen1 → gen2 → gen3 → gen4 → gen5 ═══   （共线：共线性保守）

基因组 C:  ═══ gen1 → gen2 → gen5 ← gen4 ← gen3 ═══   （检测到倒位）

基因组 D:  ═══ gen1 → gen2 → genX → gen3 → gen4 ═══   （genX 插入：基因组岛？）
```

重排（倒位、易位、基因组岛插入）可通过基因组**点阵图（dot plot）**（见[模块 3，第 1.2 节](../03-sequence_analysis/README.md)）或 **Mauve**、**MUMmer**、**D-Genies** 等工具检测。

## 5. 概念总结

现代基因组学并不以组装为终点。基因组的真正价值在于我们从中能提取出什么：

- **注释**将一段序列转化为基因和功能的目录；
- **变异检测**允许以单碱基分辨率比较基因组；
- **比较基因组学**揭示了在完整基因组尺度上的多样性、适应性和进化。

在本模块中你学到了：

- 程序如何**预测基因**（原核和真核），以及为什么在真核生物中更困难；
- 功能注释是基于相似性的**预测**，而非实验确定性；
- 如何**检测变异**（SNP/indel）以及如何谨慎解读；
- 什么是**泛基因组**及其如何比较多分离株的基因库；
- 如何在组装好的基因组中搜索**抗性基因、毒力因子和质粒**。

> [!IMPORTANT]
> 一个已注释的基因组不是最终产品；它是一个需要通过实验验证并随着数据库改进而更新的假设。
