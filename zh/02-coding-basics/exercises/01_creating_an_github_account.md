# 💻 练习：创建 GitHub 账户及 GitHub Codespaces 入门

## 引言

在现代生物信息学中，计算工具与传统的湿实验室工作同等重要。测序一个基因组、比较数千种蛋白质或重建一段进化历史，这些任务完全依赖于在命令行环境中执行命令、组织文件和自动化分析的能力。然而，安装和配置一个功能齐全的工作环境可能是一个重要的初始障碍，尤其是在不同的操作系统上工作时。

为了克服这一障碍，在本模块中我们将使用 **GitHub Codespaces**：一个基于云的完整开发环境，可直接从浏览器运行，无需安装额外的软件。Codespaces 提供一个功能齐全的 Unix 终端，预装了编程工具，并可从任何连接互联网的设备上工作。

但在使用 Codespaces 之前，我们需要创建一个 **GitHub** 账户，这是世界上使用最广泛的代码和项目托管平台。GitHub 不仅仅是一个代码仓库：它还是一个科学协作网络，人们在这里共享生物信息学流程、分析脚本、可复现的工作流以及研究项目数据。

在本练习中，你将学习创建 GitHub 账户、探索该平台并配置你的第一个 Codespaces 工作空间，我们将在后续课程中使用它来学习 Unix 命令行基础。

## 🔬 操作步骤

### 第 1 部分：什么是 GitHub？

**GitHub** 是一个基于云的平台，允许通过版本控制系统 **Git** 存储、管理和共享代码。在生物信息学中，它广泛用于：

- 共享和记录分析**脚本和数据处理流程（pipeline）**。
- 以可复现的方式协作进行数据分析项目。
- 访问科学界开发的工具和工作流。
- 管理项目版本（了解什么被更改、何时更改以及由谁更改）。

在 GitHub 中你会遇到的一些关键概念：

| 术语 | 描述 |
|---|---|
| **Repository (repo)** | 存储在 GitHub 上的文件夹或项目 |
| **Commit** | 对项目所做更改的记录 |
| **Branch** | 项目的并行版本，用于独立开发 |
| **Fork** | 他人仓库的个人副本 |
| **Codespace** | 云端开发环境，可直接从浏览器使用 |

### 第 2 部分：创建 GitHub 账户

#### 逐步操作：

1. **访问 URL**：https://github.com
2. **注册**：点击 **Sign up** 按钮（右上角）。
3. **输入信息**：
   - 电子邮箱地址。
   - 安全密码（至少 8 个字符，包含字母、数字和符号）。
   - 唯一的用户名（这将是你在 GitHub 上的公共身份，例如 `juanperez-bio`）。
4. **人机验证**：完成 CAPTCHA 验证。
5. **确认邮箱**：GitHub 会向你的邮箱发送一个验证码。输入它以激活账户。
6. **配置个人资料**（可选但推荐）：
   - 前往你的头像（右上角）→ `Settings`。
   - 添加你的全名、机构和头像。

> [!TIP]
> 选择一个专业的用户名。该名称将出现在你所有的公共仓库中，并且经常在学术和职业领域被用作参考。

### 第 3 部分：探索 GitHub 界面

创建账户后，花几分钟探索该平台：

- **Dashboard**（主页）：显示最近的活动、推荐的仓库和通知。
- **Explore**（[github.com/explore](https://github.com/explore)）：按主题发现热门项目。搜索术语如 `bioinformatics`、`RNA-seq` 或 `phylogenetics`。
- **你的个人资料**：可通过右上角的头像访问。这里显示你的仓库、贡献和活动。
- **仓库**：点击 `+` → `New repository` 查看如何创建新项目（暂时不需要创建）。

### 第 4 部分：访问 GitHub Codespaces

**GitHub Codespaces** 提供一个基于 **Visual Studio Code** 的完整云端开发环境，集成了 Unix 终端。在本练习中，我们将使用为本课程准备的一个示例仓库。

#### 逐步操作：

1. **访问课程仓库**：你的讲师将分享仓库链接。通过你的 GitHub 账户访问它。
2. **打开 Codespace**：
   - 在仓库中，点击绿色按钮 **`<> Code`**。
   - 选择 **`Codespaces`** 标签页。
   - 点击 **`Create codespace on main`**。
3. **等待初始化**：GitHub 会自动 fork（创建一个副本）并构建环境。首次可能需要 1 到 3 分钟。
4. **探索环境**：
   - 编辑器 **Visual Studio Code** 将在浏览器中打开。
   - 识别**文件浏览器**（左侧面板）、**代码编辑器**（中间面板）和**终端**（底部面板）。
5. **打开终端**：如果终端不可见，前往顶部菜单的 `Terminal` → `New Terminal`，或使用快捷键 `` Ctrl + ` ``。

> [!NOTE]
> 每个 Codespace 是一个独立的 Linux 容器。只要 Codespace 处于活动状态，其中创建的文件将保存在你的 GitHub 账户中。

> [!WARNING]
> **更改是永久的吗？** 不完全是。请注意以下事项：
> - 只要你关闭浏览器，**文件仍然存在**，前提是 Codespace 仍处于活动状态。
> - GitHub 会在约 30 分钟不活动后**自动停止 Codespace**（文件仍保留，但环境被"冻结"）。
> - 如果 Codespace **30 天未使用**，GitHub 会**自动删除**它，所有未保存的更改都将丢失。
> - **永久且安全地保存更改**的唯一方法是执行 `git commit` + `git push` 将更改推送到仓库。我们将在后续课程中学习这些操作。
> - 在 GitHub 免费计划中，你每月有 **120 小时**的 Codespaces 使用时间。对本课程来说绰绰有余。

### 待办：实践练习

**目标**：通过在 Unix 终端中运行第一个命令来验证 Codespaces 环境是否正常工作。

在 Codespaces 中打开终端后，逐个执行以下命令并记录每个命令的输出结果：

1. **我是谁？我在哪里？**
   ```bash
   whoami
   pwd
   ```
   - `whoami`：显示系统中当前用户的名称。
   - `pwd`（*print working directory*）：显示你当前所在目录的路径。

2. **这个目录里有什么？**
   ```bash
   ls
   ls -la
   ```
   - `ls`：列出当前目录中的文件和文件夹。
   - `ls -la`：显示带详细信息（权限、大小、日期）的列表，包括隐藏文件。

3. **我在使用什么操作系统？**
   ```bash
   uname -a
   ```
   显示操作系统和内核信息。你能识别出操作系统吗？

4. **挑战 1**：创建一个名为 `mi_primera_carpeta` 的目录并导航到该目录中：
   ```bash
   mkdir mi_primera_carpeta
   cd mi_primera_carpeta
   pwd
   ```
   `pwd` 显示的路径发生变化了吗？

5. **挑战 2**：创建一个简单的文本文件，包含你的名字和一条备注：
   ```bash
   echo "你好，我是 [你的名字]。这是我在 Unix 中创建的第一个文件。" > mi_nota.txt
   cat mi_nota.txt
   ```
   `>` 符号的作用是什么？`cat` 命令的作用是什么？

6. **挑战 3**：返回上一级目录并列出其所有内容。你创建的文件夹出现了吗？
   ```bash
   cd ..
   ls
   ```

### 📝 反思问题（练习后）

这些问题旨在让你不仅仅是创建账户，而是思考协作和可复现工作对生物信息学的影响：

**关于 GitHub 和可复现性**

1. **版本控制**：将分析脚本保存在 GitHub 上相比仅保存在本地计算机上有什么优势？想象一个你与不同城市的研究者团队合作的场景。
2. **公共仓库 vs. 私有仓库**：GitHub 允许创建公共仓库（对所有人可见）和私有仓库（仅对你自己或选定的合作者可见）。如果你正在为一篇尚未发表的科学文章开发生物信息学流程，你会使用公共仓库还是私有仓库？发表之后呢？
3. **Codespaces 和可复现性**：Codespaces 的一个优势是所有学生的工作环境完全相同，无论他们个人计算机的操作系统是什么。为什么你认为这在生物信息学中很重要，因为分析必须是可复现的？

**关于 Unix 环境**

4. **终端 vs. 图形界面**：生物信息学中使用的大多数高性能计算服务器（HPC）只能通过命令行访问，没有图形界面。根据你运行的前几个命令，你认为在终端中工作相比使用带按钮和菜单的程序有什么优势？
5. **命令 `pwd`**：你在 `cd mi_primera_carpeta` 前后执行了 `pwd`。为什么在终端中工作时始终知道自己在哪个目录很重要？如果不知道可能会出什么问题？
