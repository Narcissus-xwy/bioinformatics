# 模块 5：DNA 和 RNA 测序 — 从分子到组装基因组

## 引言

读取遗传密码的能力已经彻底改变了现代生物学。从最初的费力尝试到当今的高通量技术，核酸测序使我们能够破译完整基因组、理解微生物多样性、诊断遗传疾病并实时追踪流行病暴发。

本模块涵盖了从 DNA 分子到组装和评估的基因组的完整路径。为了理解测序技术*如何*工作，我们将首先回顾分子基础（核苷酸结构、DNA 复制）。然后探索测序平台、它们产生的数据质量，最后是如何从数百万个片段重建基因组。

尽管通常主要讨论 DNA 测序，但其中许多原则也适用于 RNA。例如在 RNA-seq 中，原始生物材料是 RNA，但在测序之前通常被转化为互补 DNA（cDNA）。在 Oxford Nanopore 等技术中，甚至可以**直接对 RNA 进行测序**。

我们将围绕七个主要部分展开：

1. **分子基础** — 核苷酸的化学和复制，以理解 Sanger 和 Illumina 为什么能工作。
2. **测序技术** — 从 Sanger 到 Nanopore：原理、误差特征及何时使用每种技术。
3. **测序数据** — FASTQ、Phred 质量分数、覆盖度（coverage）、深度（depth）和读长清洗。
4. **重建策略** — mapping vs 从头组装（*de novo*）。
5. **组装算法** — OLC、De Bruijn 图（De Bruijn graph）和 k-mer。
6. **组装评估** — N50、L50、BUSCO、污染及补充指标。
7. **工具** — 每个步骤的软件生态系统。

> [!NOTE]
> 测序不是实验工作的终点；它是生物信息学分析的起点。一个好的生物信息学家不是只会运行程序，而是理解其结果意味着什么以及为什么分析可能失败。

## 1. 测序的分子基础

在理解测序仪如何工作之前，你需要记住它在读取什么。所有测序技术都利用 DNA 的化学性质，因此简要的分子回顾将为你省去很多困惑。

### 1.1 核苷酸结构

**核苷酸（nucleotide）**— DNA 和 RNA 的基本单位 — 有三个组分：

```text
                         含氮碱基
                            │
                      ┌─────┴────┐
                      │  A/T/G/C │
                      └─────┬────┘
                            │
                            1'
 5'─ 磷酸基 ── 糖 (脱氧核糖) ─┬ 
      (PO₄)                  3'┴OH ← 这个基团是关键
```

| 组分 | 功能 |
|---|---|
| **磷酸基**（5'） | 将一个核苷酸连接到下一个 → 形成骨架 |
| **糖**（DNA 中为脱氧核糖，RNA 中为核糖） | 结构支撑；**3'-OH** 位置是连接下一个核苷酸的地方 |
| **含氮碱基** | 携带遗传信息（DNA 中为 A, T, G, C；RNA 中为 A, U, G, C） |

> [!IMPORTANT]
> 糖的 **3'-OH** 基团至关重要：它是 DNA 聚合酶在复制过程中添加下一个核苷酸的位置。**如果此基团被去除或阻断，链就不能继续生长。**这正是 Sanger 测序所利用的特点。

### 1.2 DNA 的极性：5' → 3'

DNA 链有**方向**。合成总是从 **5' 到 3'** 进行，因为新的核苷酸连接到前一个核苷酸的 3'-OH 上：

```text
合成方向 →

5'──P──糖──P──糖──P──糖──3'OH
       │          │          │
       A          T          G
       │          │          │
 3'OH──糖──P──糖──P──糖──P──5'

                  ← 互补链（反平行）
```

两条链是**反平行的（antiparallel）**：一条走 5'→3'，另一条走 3'→5'。这对理解 paired-end 读长的读取方式很重要。

### 1.3 DNA 复制（简化回顾）

复制是细胞在分裂前复制其 DNA 的过程。主要参与者是：

```text
                    复制叉
                      │
 5'═════════════════╗ │ ╔═══════════════3'
 3'═════════════════╝ │ ╚═══════════════5'
                      │
                  解旋酶
              （打开双链）

 前导链（连续）：
 5'────────────→ 3'    聚合酶在 5'→3' 方向不间断复制

 滞后链（不连续）：
 3'←────  ←────  ←──── 5'   冈崎片段，"向后"合成然后连接
```

**为什么这对测序很重要？**

- **Sanger** 使用 DNA 聚合酶在体外合成模板链的副本。ddNTP 在随机位置将其终止。
- **Illumina** 使用**桥式扩增（bridge amplification）**：锚定在固体表面上的 DNA 分子复制形成克隆*簇（cluster）*。每个簇包含一个片段的数千份相同副本，从而能够检测荧光信号。
- **PacBio** 实时观察**单个聚合酶**掺入荧光核苷酸。
- **Nanopore** 不使用聚合酶进行读取：DNA（或 RNA）分子物理上穿过纳米孔。

### 1.4 Sanger 的诀窍：双脱氧核苷酸（ddNTP）

**双脱氧核苷酸（ddNTP）**是缺乏 **3'-OH** 基团的修饰核苷酸：

```text
正常核苷酸 (dNTP):              双脱氧核苷酸 (ddNTP):

    碱基                              碱基
      │                                  │
  糖 ─ 3'─OH  ← 能使链             糖 ─ 3'─H   ← 不能使链
      │         继续生长               │           继续生长
  磷酸基                             磷酸基
```

当聚合酶掺入 ddNTP 而非正常 dNTP 时，链**在该位置终止**，因为没有 3'-OH 可以添加下一个核苷酸。由于每种 ddNTP（ddATP, ddTTP, ddGTP, ddCTP）标记有不同的荧光染料，可以读取在每个位置终止链的碱基。

> [!TIP]
> Sanger 能工作是因为**3'-OH 的缺失阻止了复制**，而每次终止都标记有不同的颜色。

## 2. 测序技术的历史

### 2.1 时间线

```text
 1977         1987         1990        2005       2006        2010      2011       2014
  │            │            │            │          │           │         │          │
Sanger      ABI 370A    人类基因组    454/Roche   Solexa/        Ion       PacBio    Oxford
(ddNTPs)    (第一台      计划        (焦磷酸)   Illumina      Torrent     (SMRT)    Nanopore
           自动测序仪)                            (SBS)          (pH)                (MinION)
```

测序通常分为三代：

| 代 | 领先技术 | 读长 | 精度 | 主要应用 |
|---|---|---|---|---|
| **第一代（Sanger）** | ABI 3730xl | 800–1,000 bp | 99.99% | 验证、单基因 |
| **第二代（NGS）** | Illumina | 50–300 bp | 99.9% | 基因组、RNA-seq、宏基因组学 |
| **第三代（长读长）** | PacBio / ONT | 10 kb – 2 Mb | 可变（Q20+） | 从头组装、结构变异 |

### 2.2 第一代：Sanger

- **原理：** 使用荧光标记的 ddNTP，在随机位置终止链。片段通过毛细管电泳分离，按大小顺序读取。
- **读长：** ~800–1,000 bp。
- **质量：** 非常高（Q > 40）。仍然是验证的参考标准。
- **应用：** 单个基因测序、克隆和质粒确认。

### 2.3 第二代（NGS）

#### 454（焦磷酸测序）— *历史性的*

- **原理：** 检测掺入核苷酸时焦磷酸的释放。
- **读长：** ~400–700 bp。
- **局限性：** 同聚物（`AAAAAA`）的高错误率。已停产。

#### Illumina（边合成边测序）

- **原理：** 在流通池上进行桥式克隆扩增 + 带有可逆终止子的荧光核苷酸。在每个循环中拍摄掺入碱基的照片。
- **读长：** 短读长，通常 *paired-end*（2×150 bp）。
- **质量：** 非常高，尽管会向读长末端下降。主要错误：替代（substitutions）。
- **应用：** 完整基因组、RNA-seq、宏基因组学、基因组监测。

### 2.4 第三代（长读长）

#### Pacific Biosciences (PacBio SMRT)

- **原理：** 固定化的聚合酶在纳米级孔（ZMW）内实时掺入荧光核苷酸。
- **读长：** 10–20 kb 或更长。在 HiFi/CCS 模式下实现高精度读长（Q30+）。
- **应用：** 从头组装、结构变异、完整转录本。

#### Oxford Nanopore Technologies (ONT)

- **原理：** DNA 或 RNA 分子穿过纳米孔；每个碱基以特征方式改变电流。
- **读长：** 超长（可达兆碱基）、便携、实时。
- **应用：** 现场快速测序、实时宏基因组学、甲基化检测。

### 2.5 Single-end vs. Paired-end

这是文库制备中的基本区别，特别是在 Illumina 中：

```text
═══════════════════════════════════════  DNA 片段 (~300-500 bp)

SINGLE-END:
→→→→→→→→→→                               只读取一端
  Read 1 (150 bp)

PAIRED-END:
→→→→→→→→→→              ←←←←←←←←←←      读取两端
  Read 1 (150 bp)         Read 2 (150 bp)
├─────────────────────────────────────┤
         已知距离 (insert size)
```

| 类型 | 产生什么 | 优势 | 局限性 |
|---|---|---|---|
| **Single-end** | 每个片段 1 条读长 | 更便宜，足以用于计数（RNA-seq） | 组装和重复区域信息较少 |
| **Paired-end** | 每个片段 2 条读长，间隔已知距离 | 更好的组装，解决重复，检测结构变异 | 成本更高，需要良好的文库制备 |

> [!TIP]
> 将 paired-end 想象成阅读一个章节的第一页和最后一页：即使你读不到中间部分，你也知道章节有多长，并能验证两页属于同一本书。

### 2.6 何时使用每种技术？

| 技术 | 优势 | 局限性 | 使用示例 |
|---|---|---|---|
| **Sanger** | 非常高的精度 | 低通量 | 验证突变、确认克隆 |
| **Illumina** | 高精度、大容量、每碱基成本低 | 短读长 | 细菌基因组、RNA-seq、宏基因组学 |
| **PacBio** | 长读长、HiFi 高质量 | 成本更高 | 复杂组装、完整异构体 |
| **Nanopore** | 超长读长、便携、实时 | 质量变异性较大 | 快速监测、宏基因组学、混合组装 |

> 如需复习 → [模块 1 → 文件格式](../01-introduction/README.md)。

### 3.2 Phred 质量分数（Q-score）

Phred 分数以对数尺度表示每个碱基的错误概率：

**Q = -10 × log₁₀(P)**

其中 `P` 是该碱基不正确的概率。

| Q | 错误概率 | 精度 | 实际意义 |
|---|---|---|---|
| 10 | 1/10 | 90% | 非常低质量 |
| 20 | 1/100 | 99% | 最低可接受阈值 |
| 30 | 1/1,000 | 99.9% | 高质量标准 |
| 40 | 1/10,000 | 99.99% | Sanger 质量 |

> [!NOTE]
> Phred 尺度是**对数**的，不是线性的。Q20 和 Q30 之间的差异不是"稍微好一点"：在 Q30 时错误比 Q20 **少 10 倍**。实际上，大多数组装工作流要求 Q ≥ 20 作为最低标准，Q ≥ 30 作为期望标准。

**🧠 理解题**

1. 一个 Q=20 的碱基错误概率为 1/100（1%）。如果你的读长有 150 个碱基，全部为 Q=20，你预期有多少个碱基是不正确的？
2. 为什么 Phred 尺度使用对数而不是简单地报告错误百分比？
3. 一台现代 Nanopore 测序仪在"simplex"模式下产生的读长平均为 Q12–Q15。这是否使数据变得无用？为什么或为什么不？
4. 单个碱基的**质量**和一条读长的**平均质量**之间有什么区别？

### 3.3 读长沿线的质量曲线

在 Illumina 读长中，质量**通常会向读长末端下降**。这是正常的，由荧光信号的渐进性不同步（*phasing*）引起：

```text
质量
(Phred)
  40 │ ██████████████████
  35 │ ████████████████████████
  30 │ ████████████████████████████████         ← Q30 阈值
  25 │ ████████████████████████████████████
  20 │ ███████████████████████████████████████████
  15 │ █████████████████████████████████████████████████
  10 │ ██████████████████████████████████████████████████████
     └──────────────────────────────────────────────────────→
       Base 1                                          Base 150
                   读长中的位置

     ◀─── 高质量 ───▶◀── 质量下降 ──▶
```

工具如 **FastQC** 和 **Falco** 自动生成此类图表。

**🔍 在位置质量图中寻找什么？**

| 你看到的 | 意味着什么 | 要做什么 |
|---|---|---|
| 读长末端质量逐渐下降 | Illumina 中因 *phasing* 而正常 | 使用 Trimmomatic 或 Fastp 切除 3' 端 |
| 第一个碱基质量突然下降（base 1） | 循环开始时信号不稳定 | 正常；有些流程忽略第一个碱基 |
| 读长中间质量严重下降 | 测序运行出现问题 | 评估数据是否可用；联系供应商 |
| 整条读长质量均匀偏低 | 样本降解、浓度不当或文库问题 | 重复提取或重新测序 |
| 特定位置出现低质量峰值 | 可能的气泡或点状伪影 | 如果是孤立的通常无害 |

> [!TIP]
> 如果质量在 **150 bp 读长中第 100 位之前**就低于 Q20，那是警告信号。如果在第 50 位之前就下降，数据可能不足以进行可靠的组装。

**🧠 理解题**

5. 看到质量图时，你如何区分运行问题和 Illumina 的正常行为？
6. 为什么 paired-end 实验中 R1 和 R2 读长的质量曲线相似很重要？
7. 给定一个图，150 bp 读长中前 120 位平均质量 >Q30，最后 30 个碱基质量下降到 Q15。使用 SLIDINGWINDOW:4:20 你会切除多少碱基？

### 3.4 覆盖度（Coverage）和深度（Depth）

这些术语有时被用作同义词，但存在区别：

- **覆盖度（Coverage）**：基因组中每个碱基被读取的**平均**次数。
- **深度（Depth）**：覆盖**特定**位置的读长数量。

```text
参考基因组：
═══════════════════════════════════════════

比对读长：
  ──────────
     ──────────
        ──────────
           ──────────
              ──────────
                 ──────────
                    ──────────
                       ──────────
                          ──────────
                             ──────────

每个位置的深度：
  1  2  3  4  5  6  6  6  6  5  4  3  2  1
  ↑                    ↑                 ↑
 低                  中心               低
(边缘)            (最大深度)           (边缘)

平均覆盖度 = 深度总和 / 基因组长度
```

30× 的覆盖度意味着平均每个位置被读取 30 次。这为区分真实变异和测序错误提供了统计置信度。

> [!NOTE]
> 高覆盖度不能纠正污染、GC 偏倚或系统性错误等问题。但它确实增加了组装和变异检测的置信度。

**🔍 我需要多少覆盖度？**

| 目标 | 推荐最低覆盖度 |
|---|---|
| 细菌从头组装（Illumina） | 30–50× |
| 高置信度 SNP 检测 | ≥ 20× |
| 宏基因组学（稀有生物） | 100× 或更高 |
| 长读长从头组装（Nanopore/PacBio） | 20–30× |
| 具有重复区域的复杂基因组 | 50–100× |

> [!TIP]
> **快速覆盖度估算：**
> ```
> 覆盖度 = (读长数量 × 平均读长长度) / 基因组大小
>
> 示例：2,000,000 条读长 × 150 bp / 2,800,000 bp (MRSA 基因组) ≈ 107×
> ```
> 如果估算的覆盖度低于 20×，在组装前评估数据是否足够。

**🧠 理解题**

8. 你有 500,000 条 150 bp 读长，来自*大肠杆菌*基因组（约 4.6 Mb）。估算覆盖度是多少？是否足以进行从头组装？
9. 为什么 contig 末端的**深度**通常低于中心？
10. 如果平均覆盖度为 60× 但某些区域只有 2×，什么可能导致这种情况？这将如何影响组装？

### 3.5 读长的 Trimming 和过滤

组装或比对之前通常需要清洗数据：

```text
BEFORE trimming:
@lectura_001
NNNNATGCGTACGTTAGCAATCGATCGATCGAATTTAACCGGTTADAPTADORADAPTADOR
     ↑                                        ↑          ↑
  N 碱基          有用的序列                低质量      残留接头
 (无数据)                                   

AFTER trimming:
@lectura_001
ATGCGTACGTTAGCAATCGATCGATCG
         ↑
   仅保留高质量区域
```

哪些被移除？

| 问题 | 要做什么 |
|---|---|
| **残留接头** | 切除末尾残留的接头序列 |
| **低质量碱基** | 切除 Q < 20 或 Q < 30 的末端 |
| **N 碱基**（未确定） | 移除或切除 |
| **过短读长** | 过滤切除后过短的读长 |

> [!WARNING]
> 并非总应该进行激进的 trimming。目标是改善质量而不消除有用信息。对于某些现代组装器（如 SPAdes），过度 trimming 可能适得其反。

**🔍 分析 QC 报告时的常见情况及如何应对**

| QC 报告中的情况（FastQC/Falco） | 可能诊断 | 建议操作 |
|---|---|---|
| ⚠️ 与接头匹配的 "Overrepresented sequences" | 残留接头 | 在 Fastp/Trimmomatic 中启用接头切除 |
| ⚠️ 末尾 "Per base sequence quality" 红色 | Illumina 信号的正常退化 | 使用 `SLIDINGWINDOW:4:20` 或 Fastp 的 `--cut_right` 切除末端 |
| ⚠️ "Per sequence GC content" 有两个峰 | 可能被另一种生物污染 | 验证培养物纯度；组装后按 %GC 检查 contig |
| ⚠️ "Per base N content" 升高 | 不确定碱基（信号模糊） | 使用 `LEADING:3 TRAILING:3` 切除 |
| ⚠️ 估算覆盖度低（<15×） | 测序的 DNA 量少 | 评估组装质量是否会好；考虑重新测序 |
| ✅ FastQC 中全部"绿色" | 良好质量的数据 | 可直接以最小 trimming 进行组装 |
| ⚠️ 细菌覆盖度非常高（>200×） | 如果使用了大量材料是正常的 | 组装可能更慢但通常更好 |

**🧠 理解题**

11. Trimming 后，数据的覆盖度从 107× 下降到 89×。这是预期中的吗？是否需要担忧？
12. Fastp 报告它删除了 8% 的读长。哪些类型的读长可能被丢弃了？
13. 为什么 trimming 保留 paired-end 读长的**配对关系**很重要？
14. 如果你对 150 bp 读长应用 `MINLEN:30`（在 Trimmomatic 中），哪种类型的读长会被此过滤器丢弃？

## 4. 基因组重建策略：mapping vs *de novo*

一旦你有了干净的读长，下一步是重建基因组信息。有两种基本策略。

### 4.1 Mapping 到参考基因组

读长被比对到已知基因组上：

```text
参考序列：  ═══════════════════════════════════════════

读长：        ──────────
                 ──────────
                    ──────────    ← 每条读长被"放置"
                       ──────────   在最佳匹配位置
                          ──────────
                             ──────────

结果：参考序列上的覆盖度图谱，
      标出变异（SNP/indel）。
```

- **优势：** 快速，当生物与已测序生物非常相似时很理想。
- **应用：** SNP/indel 检测、覆盖度分析、流行病学监测。
- **局限性：** 不检测参考序列中不存在的新序列（基因、质粒、基因组岛）。

### 4.2 从头组装（*De novo*）

基因组在**没有参考**的情况下从读长之间的重叠重建：

```text
读长（无序）：

  ──ATGCGT──    ──CGTACG──    ──TACGTTA──    ──GTTAGCA──

重叠：

  ──ATGCGT──
      ──CGTACG──
          ──TACGTTA──
              ──GTTAGCA──

结果 contig：
  ═══ATGCGTACGTTAGCA═══
```

- **优势：** 发现新序列、质粒、基因组岛。
- **应用：** 非模式生物基因组、探索性分析。
- **局限性：** 取决于质量、覆盖度、读长长度和基因组复杂度。

### 4.3 比较

| 策略 | 何时使用？ | 优势 | 局限性 |
|---|---|---|---|
| **Mapping** | 有良好的近缘参考序列 | 快速，对变异敏感 | 可能丢失新序列 |
| **De novo** | 无参考序列，或希望独立 | 发现新内容 | 计算要求更高 |

在实践中，两种方法互补：先进行*从头*组装，然后与参考序列比较。

## 5. 组装算法

组装就像从数百万片纸片重建一本书。核心问题：决定哪些碎片在一起以及按什么顺序。

### 5.1 基本概念

| 术语 | 含义 |
|---|---|
| **读长（Read）** | 被测序的 DNA 片段 |
| **Contig** | 从多条读长组装出的连续序列 |
| **Scaffold** | 排序和定向的 contig（使用 paired-end 信息） |
| **重复区域** | 在组装中产生歧义的相似序列 |

### 5.2 Overlap-Layout-Consensus (OLC)

这种方法比较读长之间的直接重叠：

```text
步骤 1 — Overlap：找到重叠的读长

  读长 A:  ──ATGCGTACG──
  读长 B:       ──GTACGTTAG──
                   ^^^^^^^^
                   重叠

步骤 2 — Layout：组织成图

  A ──────→ B ──────→ C ──────→ D

步骤 3 — Consensus：推导最终序列

  ═══ATGCGTACGTTAGCAATCG═══
```

- **何时使用：** 长读长（Sanger, PacBio, Nanopore），其中读长总数较少。
- **局限性：** 对于数百万条短读长，比较所有读长之间的重叠非常昂贵。

### 5.3 De Bruijn 图

这是组装**短读长**（Illumina）的主导策略。不比较完整读长，而是将其分成固定长度的片段，称为 **k-mer**。

#### 逐步演示

假设序列 `ATGCGTACG` 且 `k = 5`：

```text
步骤 1 — 从读长中提取 k-mer：

序列：A T G C G T A C G
           ─────               → ATGCG
             ─────             → TGCGT
               ─────           → GCGTA
                 ─────         → CGTAC
                   ─────       → GTACG

步骤 2 — 创建 (k-1)-mer 节点并连接：

每个 k-mer "ATGCG" 连接 (k-1)-mer "ATGC" 与 "TGCG"：

  ATGC → TGCG → GCGT → CGTA → GTAC → TACG

步骤 3 — 遍历图以获得 contig：

  ═══ ATGCGTACG ═══
```

#### 为什么对短读长有效？

因为它避免了"全部-对-全部"比较。将数百万条读长的信息概括为一个紧凑的图。

#### k 值大小的重要性

```text
小 k（例如 k=21）：
┌──────────────────────────────┐
│ + 更多连接                   │
│ + 耐受较低覆盖度             │
│ - 混合重复区域               │
│ - 组装更模糊                 │
└──────────────────────────────┘

大 k（例如 k=77）：
┌──────────────────────────────┐
│ + 更好地解决重复             │
│ + Contig 更特异              │
│ - 需要更高覆盖度             │
│ - 对测序错误敏感             │
└──────────────────────────────┘
```

> [!TIP]
> 想象你想用字母碎片重建一个句子。如果碎片非常小，许多组合看起来都可能。如果它们非常大，任何拼写错误都会破坏连接。组装必须找到平衡。

### 5.4 组装中的常见问题

| 问题 | 影响 | 常用解决方案 |
|---|---|---|
| **低覆盖度** | 无证据的间隙 | 测更多或使用长读长 |
| **测序错误** | 假 k-mer，图中的"气泡" | 组装前的纠错 |
| **重复区域** | 图中的歧义路径 | 长读长或大插入片段的 paired-end |
| **污染** | 混杂的外来序列 | 读长过滤，GC 分析 |
| **多个复制子** | 质粒作为独立 contig 组装 | 组装后评估，环化 |

## 6. 组装评估

一个组装不应仅因为产生了 contig 就被接受。必须在多个维度上进行批判性评估。

### 6.1 连续性指标：N50 和 L50

#### N50 — 逐步演示

假设你组装了一个基因组并获得了 6 个 contig：

```text
步骤 1 — 按从大到小排序：

  Contig 3:  ══════════════════════════  (50 kb)
  Contig 1:  ════════════════════        (40 kb)
  Contig 5:  ═══════════════             (30 kb)
  Contig 2:  ══════════                  (20 kb)
  Contig 6:  ═══════                     (15 kb)
  Contig 4:  ═══                         ( 5 kb)
                                         ───────
                              总计：      160 kb

步骤 2 — 计算总大小的 50%：

  160 kb 的 50% = 80 kb

步骤 3 — 从最大 contig 开始累加直至超过 50%：

  Contig 3:  50 kb  →  累计 =  50 kb  (未达到 80)
  Contig 1:  40 kb  →  累计 =  90 kb  (超过 80!)  ← 这个
                                                     ↑
                                                   N50 = 40 kb
```

**N50 = 40 kb** → "越过"50% 阈值的 contig。

#### L50

达到 50% 所需的**最少 contig 数量**：

```text
  Contig 3 (50 kb) + Contig 1 (40 kb) = 90 kb ≥ 80 kb

  → L50 = 2  (只需要 2 个 contig)
```

#### NG50 和 LG50

比标准 N50/L50 更稳健。它们使用**估算的基因组大小**而非总组装大小作为参考。这避免了不完整的组装看起来具有"良好"N50 的情况。

| 指标 | 参考 | 测量什么 |
|---|---|---|
| **N50** | 总组装大小 | 相对于组装本身的连续性 |
| **NG50** | 预期基因组大小 | 相对于预期真实基因组的连续性 |
| **L50** | 组装的 50% | 覆盖一半所需的最少 contig 数 |
| **LG50** | 预期基因组的 50% | 同上，但以预期基因组大小为基准 |

### 6.2 完整性指标

一个组装可能 N50 很高但丢失了一半的基因。

#### BUSCO

**BUSCO**（*Benchmarking Universal Single-Copy Orthologs*）搜索在任何被评估谱系生物中都应该存在的"核心"基因：

```text
典型 BUSCO 结果：

  ████████████████████████████████░░░░░  (95% Complete)
  ██                                     ( 1% Duplicated)
  ░░                                     ( 2% Fragmented)
  ░░                                     ( 2% Missing)

  C:95.0%  [S:94.0%, D:1.0%]  F:2.0%  M:2.0%  n:452
```

| 类别 | 含义 |
|---|---|
| **Complete (C)** | 基因以完整形式存在 |
| **Single-copy (S)** | 单一拷贝（预期如此） |
| **Duplicated (D)** | 多于一份拷贝 → 真实重复或伪影 |
| **Fragmented (F)** | 基因不完整 |
| **Missing (M)** | 未找到该基因 |

#### 总大小 vs. 预期大小

如果你期望一个约 4.5 Mb 的细菌基因组但得到 3.0 Mb，某些地方出错了（覆盖度不足、污染或文库制备中的丢失）。

### 6.3 准确性和一致性指标

| 指标 | 评估什么 | 工具 |
|---|---|---|
| **Mapping rate** | 原始读长中有多少百分比比对到组装？(>95% 是好的) | BWA + samtools |
| **QV**（*Quality Value*） | 碱基级精度（QV ≥ 40 = 优秀） | Merqury |
| **k-mer 频谱** | 无参考的完整性和重复评估 | Merqury, KAT |

### 6.4 纯度（污染）指标

在微生物学中这至关重要：

```text
Contig GC 含量分布：

  Contigs
    │
    │      ████
    │     ██████
    │    ████████        ██
    │   ██████████      ████
    │  ████████████    ██████
    └──────────────────────────→  %GC
       30%  35%  40%   55%  60%
       ↑                  ↑
   主要生物          污染？

如果有两个 GC 峰 → 可能被污染
```

### 6.5 批判性解读

一个好的组装**不是**N50 最高的那个，而是合理结合了以下方面的那个：

- **连续性（continuity）**（少量长 contig）；
- **完整性（completeness）**（所有预期基因都存在）；
- **准确性（accuracy）**（碱基正确，良好的 mapping 率）；
- **纯度（purity）**（无污染）；
- **大小与预期一致**。

> [!IMPORTANT]
> 对于检测抗性基因，一个碎片化但注释良好的组装可能就够了。对于研究染色体重排，需要更连续的结构。**所需质量取决于生物学问题。**

### 7.1 质量控制（QC）

| 工具 | 功能 |
|---|---|
| **FastQC** | 质量指标可视化：每碱基 Q、GC 含量、重复、接头 |
| **Falco** | FastQC 的更快速替代方案 |
| **MultiQC** | 将多个报告整合为摘要视图 |

### 7.2 清洗和处理

| 工具 | 功能 |
|---|---|

| **Trimmomatic** | 经典工具：去除接头和低质量碱基 |
| **Cutadapt** | 专门用于去除接头 |

### 7.3 比对（Mapping）

| 工具 | 读长类型 |
|---|---|
| **BWA / Bowtie2** | 短读长（Illumina） |
| **Minimap2** | 长读长（PacBio, Nanopore）以及短读长 |

### 7.4 从头组装（*De novo*）

| 工具 | 类型 | 备注 |
|---|---|---|
| **Velvet** | De Bruijn 图 | 经典，适合理解 k-mer 效应 |
| **SPAdes / Shovill** | De Bruijn + 校正 | 使用 Illumina 进行细菌基因组组装的标准 |
| **Canu / Flye** | OLC 及变体 | 用于长读长（PacBio, Nanopore） |

### 7.5 组装评估

| **Merqury** | QV 和 k-mer 频谱（无参考） |
| **CheckM** | 原核生物基因组的完整性和污染 |

### 7.6 可视化

| 工具 | 功能 |
|---|---|
| **IGV**（*Integrative Genomics Viewer*） | 比对、覆盖度和变异可视化 |
| **Tablet** | 组装和比对读长的视觉探索 |
| **Bandage** | 组装图的可视化 |

## 8. 概念总结

本模块涵盖了从分子到组装基因组的完整路径。你学到了：

- 核苷酸化学（特别是 **3'-OH**）解释了为什么 Sanger 测序能工作；
- 测序技术在读长、精度、成本和应用上有所不同；
- **Phred 质量**和 **FASTQ 格式**是报告原始数据的语言；
- **覆盖度（coverage）**和**深度（depth）**决定分析的统计置信度；
- 组装可以通过参考序列（**mapping**）或独立进行（***de novo***）；
- **De Bruijn 图**和 **k-mer** 是现代短读长组装器的基础；
- 组装的**质量**使用**多个指标**来评估（N50, BUSCO, mapping 率, 纯度）。

在**模块 6**（基因组学）中，你将使用这些组装基因组并赋予它们生物学意义：基因注释、功能预测、变异检测和比较基因组学。

> [!IMPORTANT]
> 现代测序不仅关乎生成数据，还关乎理解数据代表什么以及它们从实验室到计算分析过程中带有哪些局限性。技术的选择、质量评估和组装策略始终取决于生物学问题。
